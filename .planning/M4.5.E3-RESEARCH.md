## Key findings

### 1. PRIVACY.md is NOT table-stakes at peer scale

**Domain researcher** — "Tools that genuinely send nothing typically have NO PRIVACY.md — esbuild, Prettier, husky have none. Astro/Next.js have PRIVACY because they DO send telemetry (and use the doc to justify opt-out flow). The 'no phone home' claim is usually made by *omission* (no PRIVACY.md, no opt-out flag exists) rather than affirmation."

**Prior art researcher** — "0 of 6 ship PRIVACY.md. Not table stakes at peer scale. Skip for E3." (Sample: GSD, Agent Skills, superpowers, planning-with-files, gsd-skill-creator, anthropics/claude-code.)

**Implication:** Shipping PRIVACY.md actively *signals* that telemetry might be a concern. Silence is more credible. The audit script + a short README "Privacy & telemetry" section achieves the same verifiability without the over-claim.

### 2. SECURITY.md shape — two viable patterns; recommend Standard

**Domain researcher** — "Two shapes dominate, both very short."

- **Minimalist** (Vitest, ~5 lines): one section "Reporting a Vulnerability" pointing to GitHub's private advisory URL `/security`. No supported-versions table. No SLA.
- **Standard** (Bun, Tauri, ~25 lines): two sections — "Supported Versions" (simple table with check/X), "Reporting a Vulnerability." Pre-1.0 convention: table row `< 1.0 — ❌` or `main branch only — ✅`. SLAs at solo scale are soft ("we will acknowledge") or omitted.

**GitHub rendering quirk** (risk researcher): use exact H1 `# Security Policy` and place at repo root. GitHub's Security tab depends on this.

**Implication:** Standard shape (Bun/Tauri pattern) is the right altitude for Signal. Cheap to write, GitHub recognizes it, and a peer who finds a security issue knows where to send it without asking.

### 3. README "Privacy & telemetry" mini-section is rare in the wild

**Domain researcher** — "Surprisingly rare as a dedicated mini-section. When present, the pattern is 2-4 lines stating the behavior + linking to a longer doc — e.g., Next.js README's Telemetry section is ~3 lines pointing at `nextjs.org/telemetry`. No strong convention exists for 'we send nothing' blurbs at this length; this is a doc gap in the ecosystem."

**Implication:** Signal can do this and it'll be a small differentiator without violating any norm. ~6 lines is fine. Pair with the audit script reference; that's the credibility lever.

### 4. README "Requirements & compatibility" — single sentence beats table at solo scale

**Domain researcher** — "Granularity is low. Most CLI READMEs state runtime version inline ('Node 18+') rather than tabulating. uv uses a one-line Highlights bullet; Vitest states 'Vite >= v5.0.0 and Node >= v18.0.0' as a single sentence under Installation. Tables appear only when the matrix is non-trivial (multiple peer deps × multiple platforms). Solo-tool convention: single sentence, not a table."

**Implication:** Don't over-engineer. A 4-row table is borderline overshoot; nesting the requirements as a `###` inside the existing `## Install` section is more aligned with ecosystem norms. **However:** Signal's compat *will* eventually grow (Linux/WSL rows when testers volunteer), so a table is defensible if kept simple. Recommend: nest a small `### Requirements & compatibility` table inside `## Install` rather than promote to top-level.

### 5. Codebase has zero `.sh` precedent in `tools/`

**Codebase researcher** — "There is no `.sh` precedent in `tools/`. All entries are Node ESM `.js`. Shell scripts exist only at `hooks/session-start.sh` — `hooks/` not `tools/`. The validator, the tests, and `measure-phase-costs.js` all assume Node. A `.sh` would be the only outlier and would need cross-platform care (E1 install matrix targets Linux + WSL). A Node version inherits the shebang, ESM, lib-extraction, and exit-code conventions for free."

**Risk researcher** corroborates — "Shell portability: bash vs zsh vs sh; macOS BSD grep vs GNU grep flag differences. What's the safest portable subset for `audit-network-calls.sh`?" Avoiding this question by writing Node is cleanest.

**Conventions for the Node version** (from `tools/validate-plugin.js` and `tools/measure-phase-costs.js`):
- Shebang `#!/usr/bin/env node`
- ESM imports (`node:fs`, `node:path`, `node:url`)
- `__dirname` via `fileURLToPath(import.meta.url)`, `ROOT = join(__dirname, '..')`
- Zero-arg "run-and-report" CLI shape; exit 1 on violations, 0 default
- Output: `console.log` for human-readable; `✗` for errors, `⚠` for warnings
- Logic extracted to `tools/lib/*.js` when reusable

### 6. `tests/helpers/template-lint.js` is the reuse target for the consistency check

**Codebase researcher** — full breakdown of the pattern from E7. Tiny zero-dep lint helper with `loadTemplate(relPath)` + `findX(...)` finder functions that return violation arrays. Each vitest test calls a finder and asserts `expect(violations).toEqual([])` with formatted messages interpolating violations.

**Implication:** E3's cross-file consistency check should add 1-3 functions to `tests/helpers/template-lint.js` (or a sibling helper file if scope is materially different). New finders: `findMissingFact(content, factPattern)`, `findJargonHits(content, jargonSet)`. Tests live alongside `synthesizer-regression.test.js` patterns.

### 7. Hard-coded fact strings are a maintenance time-bomb (the bug Brett just hit)

**Risk researcher** — "Symptom: Consistency test breaks on every Epic close when test count moves 384→387→392. Trigger: Hard-coded '384 tests' string in 3 files + test asserting equality. Mitigation: Don't hard-code the number anywhere. Test reads test count from `package.json` or a single `references/facts.md` source-of-truth, then asserts README/PRIVACY/SECURITY all match that value. Same pattern for Node version + dep count. One file changes per fact, test catches drift."

**Implication:** Add `references/facts.md` as canonical source. Test reads from it, asserts referenced docs contain the same strings. Drift is now a single-file edit + the test catches it. This is also the fix to the staleness class Brett observed earlier in this session (test count was 93+ in README, 366 in CONTEXT.md, until Brett's Cursor commit refreshed both to 380+/384 manually).

### 8. CHANGELOG.md append pattern — under existing `[0.1.3]` H2

**Codebase researcher** — "Add new `###` subsections under the existing L9 `## [0.1.3]` heading. Do not create a new `##` header. E7 already shipped 4 subsections (`### Fixed`, `### Added`, `### Changed`, `### Added — docs/install-troubleshooting.md`); the pattern allows multiple `### Added` blocks distinguished by em-dash topic suffix."

**Implication:** E3 appends 2-3 new `###` subsections inside the existing `[0.1.3]` H2, after E7's blocks, before the `[0.1.2]` boundary. No new version header.

### 9. README insertion points (codebase researcher recommendations)

- **"Privacy & telemetry" (S1):** insert as new `##` between `## .planning/ is your project's memory...` (L162-166) and `## Command reference` (L168). Both are short policy/trust statements; reader sees them before the command catalog.
- **"Requirements & compatibility" (S2):** two options — (a) new `##` between `## Why this exists` (L21) and `## Install` (L23); (b) nest as `### Requirements & compatibility` inside `## Install` (L23-24). **Option (b) is less disruptive to existing flow and aligns with the "single sentence beats a table" ecosystem norm.**

### 10. Audit script scope + portability

**Risk researcher** — concrete recommendations for the audit script (now Node):

- **Patterns to grep**: `fetch(`, `axios`, `node-fetch`, `got`, `http.request`, `https.request`, `require\(['"\`](http|https|node-fetch|axios|got)`, `import.*from.*['"](http|https|node-fetch|axios)`, `child_process.*(curl|wget)`, `\.get\(['"]http`.
- **Scope (include)**: `.js`, `.sh`, `.json` in `tools/`, `skills/`, `agents/`, `commands/`.
- **Scope (exclude)**: `node_modules/`, `tests/`, `.planning/`, `analysis/`, `*.md`, plus the audit script's own path (self-reference).
- **Document scope in script header** so future edits don't widen accidentally.
- **CI enforcement**: wire into `npm run validate` AND a vitest wrapper. Pre-commit is overkill at solo scale; `npm test` is enough.
- **Disclaim transitive deps** in PRIVACY messaging — audit covers Signal's own source, not its dependency tree.

### 11. Jargon-lint anchor for SECURITY.md

**Risk researcher** — "Bare-word grep false-positives on 'the phase of the moon' or 'tier-1 priority.' Anchor lint to capitalized workflow terms + slash-command syntax: grep for `\b(Tier|Phase|Slice|Wave|Epic|Milestone)\b` (capital-initial only) and `/sig:` literal. SECURITY.md is short enough that any capitalized 'Phase' is workflow vocab, not English."

**Implication:** Jargon-lint regex is `/\b(Tier|Phase|Slice|Wave|Epic|Milestone)\b|\/sig:/`. Case-sensitive on the workflow terms; literal `/sig:` for command references.

## Surfaced decisions (require user lock before EXECUTE)

The following 3 are deviations from the locked DISCUSS scope based on research. Each REDUCES work AND improves quality. Surfacing per planner's "Surface, don't silently resolve" discipline.

### D-E3-1-amend — Drop PRIVACY.md as a separate file

**Was:** README "Privacy & telemetry" section + dedicated `PRIVACY.md` + `tools/audit-network-calls.sh`.

**Recommend:** README "Privacy & telemetry" section + `tools/audit-network-calls.js`. **No PRIVACY.md.**

**Rationale:** 0 of 6 peer plugins ship one. Domain norm is silence-as-no-telemetry. Shipping PRIVACY.md actively signals "maybe there's telemetry to worry about" — opposite of intent. The README mini-section (with audit script reference) carries the affirmative claim; the script makes it verifiable.

**Impact:** AC2 (PRIVACY.md exists) drops. AC1 (README Privacy section) absorbs the audit-method content as a short paragraph. AC3 (audit script) unchanged.

### D-E3-1-amend-b — Convert `tools/audit-network-calls.sh` → `.js`

**Was:** `tools/audit-network-calls.sh`.

**Recommend:** `tools/audit-network-calls.js` (Node ESM).

**Rationale:** Zero `.sh` precedent in `tools/`. All 15 existing files are Node ESM. Avoids shell portability question entirely (bash vs zsh, BSD vs GNU grep). Inherits Signal's standard tool conventions (shebang `#!/usr/bin/env node`, ESM, exit codes).

**Impact:** AC3 unchanged in effect; file extension + implementation language change. README + CHANGELOG references update from `.sh` to `.js`.

### D-E3-NEW-13 — Add `references/facts.md` as canonical source-of-truth

**Was not in DISCUSS scope.** Surfaces from risk research.

**Recommend:** Add `references/facts.md` with canonical strings for Node version, Claude Code version floor, runtime dep count, test count, license, repo URL. Cross-file consistency test sources from this file; README + SECURITY.md treat it as authoritative. Drift becomes a single-file edit + the test catches mismatch.

**Rationale:** Hard-coded fact strings across 3 docs = the maintenance time-bomb that just hit you on STATE.md staleness. Single source-of-truth eliminates the class.

**Impact:** New file `references/facts.md`. S2.t1 creates it. S2 consistency test sources from it. Approximately +30 LOC of new content (the facts file) + slightly more robust test logic.
