## What changed mid-flight

- **The entire S2–S5 plan was rewritten (2026-05-30).** The 2026-05-14 draft proposed destination heuristics (`suggestDestination`) and an `[include/defer/cull]` drain. The re-plan **cut heuristics** (Decision 5 — routing is explicit flags or the default FUTURE-IDEAS, nothing in between) and reshaped the drain to **advisory promote/defer/merge/delete** (Decision 2/3). This was the right call surfaced by DISCUSS, not an EXECUTE-time scramble — but it's a large mid-Epic pivot worth noting: Slice 1 shipped against a plan that no longer describes Slices 2–5.
- **Q2 refined a second time, at REVIEW (this session, `ae00a50`).** PLAN's Q2 locked "dispositioned = entry whose Status contains a disposition verb." REVIEW found that bare-word rule hid a genuine live entry (see below). The user approved tightening it to match only the drain's own stamp signature (`{Verb} {date} (… drain)`). A locked decision was refined — deliberately, with explicit user sign-off, not silently.
- **Drain "integration" tests became command-spec-contract tests.** VALIDATION's Nyquist map listed FR7.1–7.4 as "integration (plan drain)." Signal has no command-execution harness for slash-command markdown (the standing OPEN-QUESTIONS item), so the drain flow is verified at two real layers instead: the JS orchestration (`applyDispositionToFile` over a temp-dir fixture) and the `plan.md` spec contract (content tests). Documented in the VERIFICATION report rather than papered over.

## What assumptions broke

- **A: "a simple bare-verb regex is precise enough for dispositioned-detection" — false.** `/\b(Promoted|Deferred|Merged|Deleted)\b/` over-matched prose: `FUTURE-IDEAS.md:224` ("**Deferred from M4.5.E7** to FUTURE-IDEAS…") is a live idea, not a disposed one, and got hidden — 1 of 29 entries. The fix (stamp-signature match) is small; the lesson is that the rule should have been run against the **real file** at PLAN, not just reasoned about. (Same lesson E8 learned about empirical API probes.)
- **B: `insertAboveFooter` appends near EOF — false on the live file.** FUTURE-IDEAS.md's only `*Last updated:*` footer has drifted to mid-file (~L1055) because entries were hand-added below it over time. RESEARCH § R1 flagged this; it bit when the S4 follow-up capture landed mid-file (above the orphaned footer) instead of at the bottom. The entry is well-formed and surfaces correctly in the drain, but the placement is wrong. Latent `/sig:add` bug, not a drain bug.
- **C: my own content-contract test regexes were right — two were not.** S5.t4's README assertion used `[^.\n]*` which stopped at the `.` in `.planning/`; the CHANGELOG assertion's `indexOf` matched a cross-reference line, not the section header. The docs were correct; the tests were too strict. Caught at red→green. Content-contract tests need the same scrutiny as the code they pin.

## What surprised us

- **The real FUTURE-IDEAS.md (29 entries) was a free integration fixture.** The parser's fence-awareness and orphaned-mid-file-footer tolerance got exercised against real, messy data the moment the drain ran — 28 candidates surfaced correctly, the one `✓ SHIPPED` heading correctly excluded. Bigger than any synthetic fixture.
- **The byte-range design made "defer all remaining" nearly free.** Because each disposition edits only its own byte range and ranges tile gap-free, a batch applied in descending index order is correct without re-parsing. `applyDispositions` fell out of `applyDisposition` with almost no extra logic.
- **Reusing the S1 capture engine for the S4 follow-up dogfooded a latent bug.** Filing the S4 note via `captureToFutureIdeas` (rather than hand-editing) is exactly what exposed assumption B — using the real code path surfaced the footer-drift problem a hand-edit would have hidden.

## What we'd do differently

- **Run candidate detection rules against the real data at PLAN.** Both the Q2 over-match (A) and the footer drift (B) were discoverable by running the proposed logic against the live `FUTURE-IDEAS.md` before locking. A one-command empirical check at PLAN would have caught both.
- **Name command-markdown test types honestly in VALIDATION.** Calling spec-contract tests "integration" forced VERIFY to reconcile a mismatch. Use "spec-contract" as a first-class test type for command-markdown features so the plan and the reality agree up front.
- **Re-validate a "locked simple rule" when it first touches real data.** Q2's simplification was reasonable in the abstract and wrong against the real file. A locked decision is not exempt from a sanity check at the moment it meets production data.

## What to feed back into Signal

- **`/sig:add` footer placement is buggy on drifted files** (assumption B). `insertAboveFooter` should target the *last* footer or true EOF, or a hygiene pass should re-anchor the FUTURE-IDEAS footer. Worth its own FUTURE-IDEAS entry / small fix.
- **A command-markdown test harness is now overdue.** Three features (`/sig:doctor`, `/sig:checkpoint`, `/sig:plan` drain) are verified by bespoke content-contract assertions. A shared helper (assert a command file contains a section with required anchors) would cut per-feature boilerplate. This is the standing OPEN-QUESTIONS item; it has now earned its keep.
- **Bake the working-norm corrections into command output.** Two corrections landed this session and are saved to auto-memory: (1) plain language in *all* output, not just questions; (2) every observation carries a recommendation or options, never a bare "heads up." Signal's own REVIEW findings and `AskUserQuestion` framings should follow both.

## Anti-rationalization moment

At REVIEW, the Q2 over-match was doubly tempting to bury: *"Q2 is a locked decision, not my place to change"* **and** *"it's an advisory feature, 1 of 29 entries, just ship it."* Both are real arguments. The discipline that mattered: **surface it with options and a point of view rather than ship the known-imperfect rule silently** — and **do not unilaterally override the locked decision**, but hand the user the choice. The user chose to fix it. Had the finding been buried under "PASS, locked decision," a real live idea would have stayed invisible on every future drain.
