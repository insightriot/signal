## 1. The command spans TWO move-classes (reshapes the whole plan)

The migrate command is **not** "three narrative vectors + a gate." It is **two distinct move-classes**, each with an existing prototype:

| Move-class | What moves | Engine today | Link risk |
|---|---|---|---|
| **Within-STATE narrative** (vectors 1/2/3) | prose *inside* STATE.md → body / `STATE-HISTORY.md` / `archive/<epic>/` + a scalar/pointer | `tools/lib/evict.js` (`evictEpicNarrative` :312, gate `verifyCardCoverage` :123) | near-zero |
| **Whole-file archive-tree** | closed-Epic scaffolding files → `archive/<m>/<e>/` + rewrite ~99 bare-identifier refs | `tools/archive-migrate.mjs` (LINK pass :60-81, PROSE pass :44, verify :116-133) | **high** (why FR6.3 exists) |

Both are required (FR6 = "reorganize to the FR1 model"). `evict.js` can't build the archive tree; `archive-migrate.mjs` can't de-prose frontmatter. The command shell wires them.

## 3. The faithfulness gate — reusable contract (FR6.7)

- **`verifyFaithful(source, card, opts)`** = an **alias/re-export of `verifyCardCoverage`** (NOT a rename — renaming churns `evictEpicNarrative:338` + tests). Called **once per relocation**; pure; `pass:false` **blocks that relocation from apply**.
- **`relocateFaithful({sourceText, card, destAbs, pointer, dropped, baseDir})`** = NEW, extracted from `evictEpicNarrative`'s spine (`evict.js:343-391`) minus Epic/retro coupling: verify → path-confine → byte-identical write + read-back assert → return pointer.
- **`dropped` starts empty**; a gate failure is **surfaced in the dry-run for the human to resolve at confirm** (fix the card, or explicitly acknowledge the drop) — **never auto-bypassed**. This converts the B8 delete-catastrophe into a blocked-and-flagged apply (removed prose → `source` tokens, empty new home → `card`; gate fails every token → apply refused).
- **Move-class nuance:** vectors 1 & 2 are **relocate-verbatim** (new home ⊇ source → gate passes by construction; residual risk = the scalar/pointer left behind + dangling links). Vector 3 is **distill-then-relocate** (gate does real work; the card **is** the pre-existing retro per §4 — never auto-author one; `not-closed` → flag, don't fabricate).
- **§5 blind spot (load-bearing NFR):** the gate proves **no token loss**; byte-identical proves **no info loss**; **neither proves semantic faithfulness**. Proof = **human-approved dry-run diff + Signal-own dogfood + REVIEW**, NEVER green tests (D-M5E2-4).

## 4. Conservative auto-sense algorithm (FR6.5)

`CURRENT_LAYOUT_VERSION = 2` (absent/`1` = pre-reorg).
1. **Read** STATE via `parseFrontmatter`.
2. **Stamp gate (fast path):** `docs_layout_version >= 2` + structural conformance → `{conformant, no-op}` (FR6.4). Else proceed.
3. **Per-vector structural sniff (each independent):**
   - **V1** — `checkStateFrontmatterShape`. `block:true` = strong positive. ⚠ **`block:false` does NOT prove clean** (it's a blacklist w/ generous 150/500-char budgets) — flag-for-review at a *lower* bar (e.g. >~80 chars / multi-clause) but **never auto-move below the block threshold**.
   - **V2** — PRIMARY = **body-size threshold AND absence of the FR2c skeleton headings** (`## Resume pointer` / `## In-flight` / `## Closed work`, `state-schema.md:81/85/99`). Marker only corroborates (the old inline-marker string isn't reliably grep-able across versions).
   - **V3** — glob `*-RETROSPECTIVE.md` → closed-Epic IDs → `extractEpicSection` found AND not already pointer-form.
4. **Ambiguity flagging:** borderline signals → `ambiguous`, plan **smallest safe move (or none)**, surface in dry-run. Only unambiguously-closed content is auto-planned.
5. **Idempotency:** stamp written **only on full conformance**; **per-vector-sniff IS the guard** (no separate migrated-flag needed) → safe re-run after a partial/interrupted run.

## 5. Ranked risks → required mitigations + tests

| **FM2** | Faithfulness (paraphrase) — vector-3 only | High·Med | dry-run renders **full source + destination + dropped-list**; **dry-run→apply hash binding** (TOCTOU: apply recomputes disk — abort if inputs changed since approval) | confirm-UX completeness test (no relocation applied without content surfaced); hash-binding abort test; **faithfulness proven only by S4 dogfood + REVIEW** |
| **FM3** | Auto-sense false pos/neg on unseen projects | High·High (dry-run-visible) | stamp-first; per-file `{vector,confidence,action\|flag}`; act only on explicit signals; flag ambiguity | fixture matrix: conformant→no-op, partial→residual-only, linear→v1/v2-only, non-standard→flag, **"shouldn't touch" append-log (DECISIONS.md)→untouched** |
| **FM7** | Link/anchor dangling | High·Med-Hi | prototype verify is **advisory** (logs after commit) → make it a **BLOCKING abort+rollback gate**; **anchor links** (`file#heading`) unverified → resolve or flag-not-rewrite; POSIX `/` on Windows | pre/post dangling count (file+anchor)=0; mutation test: skip a referrer rewrite → blocking verify aborts |

**Cross-cutting (finding B):** **operate on raw frontmatter lines — never round-trip through `stringifyYaml`.** One decision de-risks FM1/FM2/FM4/FM8 at once (a full re-serialize reorders keys, buries the real change in the review diff, breaks idempotency, and forces a full YAML parse of a 529 KB block).

## 6. Design patterns (prior art — borrow interaction shape, not deps)

- **Three-tier dry-run:** T1 summary counts (jscodeshift), T2 mechanical plan (moves/links — machine-checked by the dangling verify, skim-only), **T3 faithfulness diff (the approval surface, Signal-native)** — source prose + distilled card + pointer + the §5 coverage ledger (each ID/date/token ✓carried / ⤴lifted / ✗dropped); per-relocation `y/n/e` (ast-grep `--interactive`).
- **Plan as a durable editable artifact** (Prisma `--create-only`): write the dry-run plan + proposed cards somewhere reviewable, apply consumes the reviewed plan. **Plumbing:** artifact location — NOT a `.planning/` path that trips the write-guard/single-home; use a scratch or clearly-marked `.planning/.migrate/` **removed on apply**.
- **Rollback:** single atomic commit + a pre-apply lightweight tag `pre-migrate-memory-<ISO>`; `git revert`. **Reject** stash (not durable) + backup-dir (single-home violation) for the git-happy path. Dirty-tree: refuse without ack (git-filter-repo model).
- **⚠ Commit-on-behalf tension (SURFACED, planner's call):** the command runs in **other people's repos** whose norms Signal can't assume ("commit only when asked"). Two options: (a) apply → auto-commit as one revertable unit + print the `git revert` line; **(b) apply → leave changes staged-but-uncommitted + print the tag/commit instructions.** Research recommends **(b)** as the safer stranger's-repo default. **PLAN adopts (b).**
- **Mark-as-migrated primitive** (Alembic `stamp` / Django `--fake` / Flyway `baseline`): a **stamp-only path** for an already-hand-migrated project (e.g. Signal's own S5 relocate that never set a stamp) — sets `docs_layout_version` without moves **only after a structural-conformance check** (Alembic's warning: never stamp blind).

### The 6 archive-dogfood lessons (2026-06-05; standing ⚠ "re-validate at full-wiki scale — only closed docs were exercised")
1. **The cascade is the LINKS, not the moves** (68 links + 92 prose paths vs 48 trivial `git mv`).
2. **Three reference classes:** clickable `](path)` → context-relative; prose `.planning/X` → archive path; bare `` `M4.5.E6-PLAN.md` `` → valid only if prefixed names kept. **TBD:** reference-style `[a]: path` + HTML anchors never exercised (FM7 gap).
3. **ID-prefixed filenames in nested folders** avoided ~99 rewrites. TBD vs a bare-name active wiki.
4. **The keying bug (load-bearing):** link edits computed vs pre-move paths, applied by mapping **original→new**; the first apply keyed by post-move path silently skipped every moved-file link — caught **only** by the post-move dangling verify → **the verify gate is not optional**.
5. **Retros stayed in root** (`retro-index.js` parses epicId from flat filename). Moving them needs parser+renderer+test updates. Deferred.
6. **Bot-race is real** (`mps-compiler` pushed mid-session) → assume the tree moves under you: fetch + dry-run + reviewable plan + atomic apply, never a blind in-place rewrite.
