## B24 — dangling-delta re-key (FR1)

- **Root cause:** `computeDanglingDelta` (`migrate-memory.js:1503-1507`) keys on `` `${d.file}\0${d.link}` ``; the append-log evict relocates a closed DECISIONS block **and** re-roots its links (`archive-tree.js:157-180`, `computeLinkEdits` at :174 — resolution-preserving), mutating both key components → pre-existing dangle misread as migrate-introduced → `enforceNoDangling` (:1674-1695) aborts + rolls back.
- **Divergence:** `renderDryRun` (:1911-1914) lists the same link as "pre-existing" (no delta run), while apply computes the delta on post-move disk → they disagree. The fix must make apply's verdict match dry-run's.
- **The invariant to key on:** `computeLinkEdits` rebuilds the link `rel` to the **identical absolute target** (`archive-tree.js:171-174`). So the resolved repo-root-relative absolute target is stable across move + reroot — that's the delta key. Use **multiset/count** semantics (not a plain Set) so an *additional* new dangle to an already-missing file still surfaces.
- **Repro test:** `tests/migrate-dangling-baseline.test.js:162-219` asserts the *current* (wrong) abort behavior and says so verbatim (`:160-161`). PLAN inverts it.

## B26 — Epic-close STATE fallback across 3 layers (FR2)

- **Root cause (confirmed, worse than ticket):** `isEpicCloseShip` (`retrospective.js:341-358`) is 100% milestone-table-driven via `findEpicStatusRow` (:727-739). **`MILESTONE-5.md` has no E4 row at all** → `null` → skip. The blind predicate is shared by **3 layers**: Layer 1 `shipFR1Check` (:667), Layer 3 `detectDirtyExecute` (:603). Layer 2 `checkProposedStateWrite` (:379,398) is blind for an *independent* reason — it keys off a `- SHIP` line in `completed_phases` that Signal **never writes** (`references/epic-native-flow.md:27`).
- **PLAN confirmation — tier→required-phases mapping** (from `references/tier-definitions.md:118` + `PHASES` at `state.js:16`): required pre-SHIP phases = `[DISCUSS, PLAN, EXECUTE, VERIFY, REVIEW]` **minus `profile.phases_skipped`**. FULL/FEATURE → all 5; SKETCH (`[REVIEW]`) → through VERIFY; SPIKE (`[REVIEW, SHIP]`) → SHIP skipped, so the SHIP-time retro gate never runs. Derive this from `PHASES` + `phases_skipped`, do **not** hardcode. Phase-name match must tolerate the `(date)` suffix (`REVIEW (2026-07-21)`).
- **Guard:** gate the STATE fallback on **row-absence** (`findEpicStatusRow === null`), not a pure OR — a maintained table saying "pending"/"in flight" must still win for a legit per-slice ship (D-E9-5).

## B25 — FR5 read-enclosure behavioral test (FR3)

- **Confirmed:** all 6 RMW wrappers are `withStateLock(baseDir, () => XxxCore(...))` with the read inside `Core` — behavior is correct. `tests/rmw-lock.test.js` only asserts throw-under-held-lock, which a broken read-outside-lock wrapper passes identically → AC5.2 has no behavioral test.
- **Seam precedent:** the codebase already injects a test-only `renameFn` opts-seam for crash tests (`atomic-write.js:20`, threaded through `drain.js:470,598,725`). Mirror it with `_afterRead` (default no-op). 4 of the 6 Cores already forward `opts` verbatim (`applyDispositionToFile`, `promoteDrainEntry`, `evictTerminalToLedger`, `generateMilestoneMetaRetro`); **`regenerateIndex` takes no opts** (`retro-index.js:251`) and needs a one-line signature bump to `(baseDir, opts = {})`.
- **Lock is throw-on-contention, not blocking** (`file-lock.js:55`, atomic `wx` create) → the *arriving* writer fails fast; the interleaving test asserts "no lost update" via final on-disk content, not "A incorporates B."
- **Doc nit:** `M5.E4-REVIEW.md:29` says "7 wrappers"; there's a 7th (`regeneratePlanningIndex`, read-enclosed, covered indirectly). `RMW_PATHS` canonicalizes 6. Reconcile wording.

## B6 — isStateStale file-identity tightening (FR4)

- **Root cause:** `isStateStale` (`state.js:769-844`) Walk 2 derives its "genuine work" exclude from the **full** `STATE_AFFECTING_PATHS` (`:721-728`, includes `*-PLAN/PROGRESS/VERIFICATION/REVIEW.md`), so a committed phase-artifact never rolled into STATE reads as "not stale." Internal inconsistency: those files are in the *trigger* set (Walk 1) but Walk 2 swallows them.
- **Fix shape:** a **separate, smaller `BOOKKEEPING_PATHS`** (= `STATE.md` + `CONTEXT.md`, ratified) drives Walk 2's exclude; leave `STATE_AFFECTING_PATHS` unchanged (removing PLAN/PROGRESS from it would make them invisible — the opposite bug — and break `is-state-stale.test.js:136`). Count-independent, so it's correct where the reviewer's `commits.length === 1` candidate misfired (split STATE refresh).
