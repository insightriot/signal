## What changed mid-flight

- **B26 "all 3 layers" descoped to Layers 1+2 (ratified).** The T2 executor hit a wall: `isEpicCloseByState` requires `phase===SHIP`, but Layer 3 (`detectDirtyExecute`) is guarded on `phase===EXECUTE` — an empty intersection, so the fallback is a literal no-op there. It **stopped and reported** rather than fake a green Layer-3 fix. I verified the claim from primary source, confirmed the harm is fully closed by Layers 1+2, and surfaced the scope change to Brett; the resume-nudge enhancement was captured (ISSUES-INBOX) rather than dropped.
- **B24's fix deviated from the catalogue.** The `BUGS.md` wording ("key on decision-ID / original link text") doesn't work — the reroot *mutates* the link text and there's no decision-ID at the shared scanner layer. The investigation found the correct key is the **resolved absolute target** (the reroot's invariant) with multiset semantics; ratified as a deviation.

## What assumptions broke

- **CONTEXT Assumption 1 ("`computeLinkEdits` is resolution-preserving") is true for relative links but NOT absolute ones.** The security panel verified relative-link invariance empirically; the code panel's H1 hypothesis, confirmed by a path-math probe, showed an absolute-path `](/abs/foo.md)` link diverges (`resolve()` resets on absolute, `posix.join()` concatenates) → the reroot mangles it → the before/after key diverges → fail-safe false-abort. B24 fixed the relative case; the absolute case is a residual (**B28**), same class as B27.
- **"The B26 fix will gate our own SHIP at the pre-check."** It fires at `phase===SHIP`, but the `/sig:ship` FR1 pre-check (Step 0.5) runs *before* the SHIP transition (Step 5) — so on a fresh REVIEW→SHIP flow the pre-check sees `phase=REVIEW` and **skips**. Only once phase is actually SHIP does it halt. Surfaced by running the gate honestly instead of assuming (**B30**).

## What surprised us

- **Dogfooding B26 on its own SHIP both validated the fix AND found a new bug.** At `phase=SHIP` with no retro, `shipFR1Check` returned `{halt:true, NO_RETRO_FILE}` — the *exact* silent-skip B26 closes, now hard-blocking its own release. But testing it at `phase=REVIEW` first (rather than asserting it works) revealed the pre-check timing gap (B30). Running the real gate beat assuming the gate.
- **Zero false-greens — the M5.E4 paranoia paid off.** The test-integrity panel's mutation matrix broke the shipped code 12 ways and confirmed the right test goes red each time; both load-bearing discriminators (AC1.4 vs a `Set`, AC3.1 vs a real read-outside-lock) genuinely fail on the wrong impl. The security panel *ran* `applyMigrate` against adversarial fixtures rather than reading it.
- **Two executors stopped-and-reported** (T2's Layer-3 wall, T3's scope divergence + coverage-precision) instead of faking past — exactly the behavior M5.E4's retro asked for, now happening by default.

## What we'd do differently

- **Run the FR1 pre-check after the SHIP transition, or teach it "about-to-close."** B30's root cause is a command-ordering quirk: the STATE-based Epic-close detector can't fire while phase is still the pre-SHIP phase. Fix the timing so the retro gate covers a fresh flow, not only a re-invocation.
- **Verify path-resolution invariants for the awkward shapes (absolute, symlink) in DISCUSS, not REVIEW.** B24's abs-target key was validated for relative links in the investigation; the absolute-path divergence (B28) only surfaced under the adversarial panel. A "does this hold for absolute/symlink/`..` targets?" check belongs with the fix design.

## What to feed back into Signal

- **The FR1 pre-check timing (B30) is a real enforcement gap** — a STATE-based Epic-close gate whose signal only exists at `phase=SHIP` cannot fire at a Step-0.5 pre-check that precedes the Step-5 transition. Either the SHIP command transitions before its own gate, or the detector recognizes "current phase is the last pre-SHIP phase, all others complete."
- **The B24/B27/B28 cluster is a design signal:** the migrate dangling gate over-aborts on *several* link shapes inside evicted/archived closed blocks (R7 inline links, absolute paths). A single decision — treat closed-block/archive links as *flag-not-abort*, matching R7's prose exemption — would dissolve the whole cluster.
- **EXECUTE guidance should distinguish "logically parallel" from "safe to dispatch concurrently."** The plan marked waves "can run in parallel" (dependency-independence), but every executor runs the full suite + commits against a shared index, so concurrent dispatch races. Sequential (or worktree-isolated) is the safe reading; the guidance should say so.

## Anti-rationalization moment

**The B26 dogfood.** I had already written most of this Epic and *knew* M5.E5 was an Epic-close — the easy path was to write the retro, assert "the B26 fix now gates SHIP," and cut the release. Instead I ran `shipFR1Check` empirically against the real state. At `phase=REVIEW` it returned `{skipped:true}` — the fix I'd just shipped, verified, and adversarially reviewed did **not** fire at the pre-check. Only by refusing to assume-the-gate did I find that the STATE fallback needs `phase=SHIP` and the pre-check runs before the transition (B30). A clean "the fix works, dogfood confirms" line would have shipped a real residual as a success story. Running the gate instead of narrating it is the moment.
