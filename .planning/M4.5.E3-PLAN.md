**Goal:** Implement the script so S1.t1 tests turn GREEN.

**Acceptance:**
- New file `tools/audit-network-calls.js`. Conventions per existing `tools/validate-plugin.js`:
  - Shebang `#!/usr/bin/env node`
  - ESM imports: `node:fs`, `node:path`, `node:url`, `node:child_process` (for `execFileSync('grep', ...)` or equivalent)
  - `__dirname` via `fileURLToPath(import.meta.url)`; `ROOT = join(__dirname, '..')`
  - Exit 0 default; exit 1 on any pattern hit
  - `console.log('✗ found network call: ' + path + ':' + line)` per hit
- **Patterns** (regex, case-sensitive where syntactically meaningful):
  - `\bfetch\s*\(`
  - `\b(axios|got|node-fetch)\b`
  - `\bhttps?\.request\b`
  - `require\s*\(['"\`](https?|node-fetch|axios|got)`
  - `import\s+.+\s+from\s+['"](https?|node-fetch|axios|got)`
  - `child_process[^)]*?(curl|wget)`
- **Scope (include)**: `tools/`, `skills/`, `agents/`, `commands/` — recurse; files matching `*.js`, `*.json` only.
- **Scope (exclude)**: `node_modules/`, `tests/`, `.planning/`, `analysis/`, `*.md`, `audit-network-calls.js` itself.
- Script header documents scope explicitly (so future edits don't widen accidentally).
- File made executable: `chmod +x tools/audit-network-calls.js`.
- All 3 S1.t1 tests pass. Seeded-fail fixture lives at `tests/fixtures/audit-network-calls-seeded/with-fetch.js` (committed; gitignored from the audit script's own scope by the exclude rule for `tests/`).
- Suite: 387 → 387 (no count change; 3 RED → 3 GREEN).

**TDD posture:** Strict — make the RED tests pass without modifying them.
