# M5.E3 — Research (4-agent parallel synthesis)

**Method:** 4 parallel researchers at FULL — (1) codebase reuse-map, (2) FR3+FR4 design, (3) FR5 append-log eviction design+risk, (4) FR1/FR2/FR6 mechanics. All findings grounded in the actual `tools/lib/*` code. This synthesizes them for the planner.

## Executive summary — the dependency graph (drives slicing)

```
FR1 (rename + smart capture)  ──►  FR6 (v2→v3 migrate)   [migrate physically renames; code must expect new names first]
      │
      └──►  FR2 (BACKLOG + drain classify/promote)  ──►  FR6   [FR6 seeds BACKLOG from snapshot]
FR3 (auto-index + D-ID→home map)  ──►  FR5 (append-log evict)  ──►  FR6   [FR5 anchor resolves ONLY via FR3's index; FR6 composes FR5]
FR3 (roster.js) ──► FR4 (all-docs hygiene, reconcile-before-guard)
```

**Load-bearing facts:**
- **FR5's risk is the *selector*, not the relocate.** Chose **verbatim relocate behind a dated pointer, NOT a distilled card** (no AC needs distillation → zero distortion risk). Selector = **date cutoff** at the current-milestone-open boundary (`2026-07-15` from `current_epic: M5.E3`→M5), evict sections strictly `<` it. Measured: `DECISIONS.md` has 45 dated sections, **38 carry no `D-` ID** → date is the only universal signal.
- **FR3's `D-ID → home` map is mechanical-fresh, so it self-heals the instant FR5 evicts.** That's what makes evict-with-anchors safe. FR3 must be **programmatically invocable** so FR5's apply regenerates it **in-transaction** (else an eviction→next-index window leaves D-IDs unresolvable).
- **FR4 lands RED today** — the live 15-vs-16 command drift (disk has 16 `commands/*.md`, CLAUDE.md/docs-map say 15). Must **reconcile all docs first, then land the guard** (can't merge a red test).
- **The rename needs a back-compat resolver (H1, mandatory).** `/plugin update` ships renamed code to every project at once, but each renames its file only when it runs `/sig:migrate-memory`. `resolveInboxPath(baseDir)` = `ISSUES-INBOX.md` if present else `FUTURE-IDEAS.md`, shared by `add.js` + `drain.js`, makes the code rename non-breaking + migrate-timing-independent.
- **The v3 stamp must be conformance-gated (H2, highest-risk in FR6).** If `CURRENT_LAYOUT_VERSION` bumps 2→3 without teaching `stampOnConformance` the v3 shape, an un-migrated project gets stamped v3 and its drift banner **goes silent**. v3-conformant ⟺ (V1/V2/V3 clean) AND `FUTURE-IDEAS.md` absent AND `BACKLOG.md` present AND FR5 evict done.

## Per-FR findings

### FR1 — ISSUES-INBOX rename + smart capture
- **Reuse:** `add.js:captureToDestination` (destination-agnostic spine), `resolveDestination`/`parseInput` (flag parser — `--bug` is a new arm alongside `--question`/`--milestone`/`--file`), `deriveHeading` (kept as the deterministic **fallback**), `scrubSensitive`/`checkBodyLength`/`acquireLock`, the OPEN-QUESTIONS `insertAtEnd` shape (closer template for `--bug` than the footer pattern).
- **New:** thread an optional `title` through `captureToDestination`→the `buildEntry` closures (`heading = title?.trim() || deriveHeading(body)`; **body stays byte-identical** — title only feeds the `## ` line); `captureToBugs`+`buildBugsEntry`; the `resolveInboxPath` resolver; the path-const/string rename.
- **Rename blast radius:** 633 raw grep → **actionable 43 prod refs across just 3 files** (`add.js` 29, `drain.js` 13, `state.js` 1 comment) + 71 test refs + 24 command-doc + a fixture dir. **Frozen (leave):** `CHANGELOG.md`, `.planning/archive/`, historical `.planning/` docs (vocab-lock 2026-05-12 precedent).
- **Decided:** **minimal** symbol rename (path const + user-facing strings + resolver; keep internal symbol names like `buildFutureIdeasEntry` — accurate, and a full rename is high-churn for zero behavioral gain, trips the `add.test.js` export-surface test). Ledger renamed `FUTURE-IDEAS-LEDGER.md`→`ISSUES-INBOX-LEDGER.md` (via the migrate moveMap). `--bug` writes a **simple entry** (heading + `**Status:** needs-triage` + verbatim body + `---`), **no B-ID allocation / no table-row** into the curated BUGS table (maintainer promotes at triage).
- **Gotcha:** `--bug` must route like `--question` (bypass the `--file` `assertSafeFilePath` denylist that blocks `DECISIONS.md`/`STATE.md`/`BUGS.md`).

### FR2 — living BACKLOG.md + drain classify/promote/evict
- **Reuse:** `drain.js:parseEntries` (fence-aware `## ` parser + byte-exact `range` + `dispositionKind`), `applyDisposition(s)`/`transformBlock` (promote/defer/merge/delete verbs), `listDrainCandidatesWithRecovery` (dangling-fence recovery), **`evictTerminalToLedger`** (crash-safe ledger-append-first→inbox-remove-second, sha1 `evictionKey` dedup) — **PROMOTED is already terminal, so AC2.4 convergence is largely already satisfied**; the `BLOCKQUOTE/STATUS/HEADING_DISPOSED_RE` triad (must keep honoring so classify doesn't resurface promoted items).
- **New (net):** the **BACKLOG live-writer** (`transformBlock` only stamps in-place; a promote must APPEND to a live `BACKLOG.md` — different entry shape, `roadmap`/`hygiene` tag) → recommend a small `tools/lib/backlog.js` with idempotent create-if-missing; the **classify axis** (work→BACKLOG / bug→BUGS, orthogonal to the disposition verbs); **retitle-on-promote** (reuses FR1's `title` threading).
- **Promote is double-homed by design** (BACKLOG = live home; ledger = frozen audit of "what left the inbox and when") — per doc-runtime-model §2, different roles, NOT a single-home violation. Mirror the crash-safe ordering: **write BACKLOG first**, then stamp+evict; sha1-dedupe the BACKLOG append.
- **BACKLOG.md structure (recommended):** reuse the FUTURE-IDEAS/OPEN-QUESTIONS entry grammar (so `parseEntries` still applies); a dedicated `**Tag:** roadmap|hygiene` line per entry (greppable, not in the heading); **sequence = file order** (never number headings — renumber churns diffs); a `*Last updated:*` footer (so promote-append reuses `insertAboveFooter`/`rewriteFooter`).
- **Snapshot migration (AC2.2):** `BACKLOG-REVIEW-2026-07-04.md` (145 lines, table-heavy) is a **judgment-driven restructure, not a mechanical move** — an agent pass lifts its live recommendations into sequenced entries, then archive the snapshot with a pointer (move-never-delete). Signal-dogfood-specific; a generic project has no snapshot → empty skeleton.
- **Light `/sig:ship` sweep (AC2.6):** `evictTerminalToLedger(dryRun)`→confirm→real at Epic-close (clears accumulated terminal entries); **not** the full classify/promote drain; `/sig:execute` never invokes it. Natural hook = the slot the retired Curator §8 vacates.

### FR3 — auto-generated `/sig:index`
- **Organizing spine:** every INDEX element is **mechanical-fresh** (recomputed from disk: roster, COLD tier, Epic-ledger status, the D-ID map) or **curated-preserve** (hand-authored, survives by file-path / Epic-ID key; absent → mechanical placeholder). Structurally identical to `retro-index.js:renderIndex(retros, existingHooks)`.
- **Reuse:** `retro-index.js:parseExistingHooks` (the exact "preserve annotation by ID" mechanism), `renderIndex`/`regenerateIndex` (render-then-compare-before-write = idempotency), `enumerateRetros`/`isStubRetro`/`epicIdCompare`, `migrate-memory.js:classifyDocGrowthPolicy`+`walkMarkdown` (enumerate+classify every doc+archive).
- **New:** `tools/lib/planning-index.js` (`enumeratePlanningDocs`/`parseExistingAnnotations`/`renderPlanningIndex`/`regeneratePlanningIndex`) — INDEX has **two keyspaces** (file-path for tier tables, Epic-ID for the ledger) + tiered tables, so a new parser/renderer per table shape; a thin `commands/index.md` (`/sig:index`); a SHIP-hook regen call **in the slot Curator §8 vacates** (beside `ship.md §6`'s RETROSPECTIVES regen).
- **AC3.3 D-ID→home map (load-bearing for FR5):** do NOT emit ~669 rows — scan **both** `DECISIONS.md` and the per-milestone history for `D-…` prefixes, emit a compact `{sourceFile → ID-prefix ranges}` map. Mechanical-fresh → **self-heals on FR5 eviction**.
- **Idempotency traps:** (1) do NOT re-derive curated notes each run (preserve by ID; absent→placeholder) — re-deriving is the clobber that made external Curator regress Signal's index (249 false entries); (2) do NOT inherit `retro-index`'s **mtime sort** — STATE.md changes every session → mtime-sort churns the diff every run and breaks AC3.4; sort file rows by a **stable key** (path/curated order). Keystone: a **parse↔render fixpoint** test (`render(parse(render(x))) === render(x)`) + the no-op-zero-diff test.
- **Prior art:** external Curator is a **cautionary tale** — its generic ID regex `[A-Z]{1,5}\d+` collides Signal IDs (A1/A01/D1) → 249 false positives + clobber-on-commit. Borrow the projection-INDEX + refresh-idempotency + read-first intent; reject generic ID detection (use Signal's own `parseEpicIdFromFilename`/`D-…` grammar), the clobber hook, any external CLI. `retro-index.js` is the real reference.

### FR4 — all-docs hygiene runtime (in the test suite)
- **Reuse:** `migrate-memory.js:scanDanglingLinks`+`walkMarkdown`+`isExternalLink` (the offline internal-link engine — broaden root past `.planning/` to `docs/`,`README`,`CLAUDE.md`,`analysis/`), `tests/cross-file-consistency.test.js:extractFact`+`references/facts.md` (the roster/count source-of-truth model), **`tools/audit-network-calls.js`+its test** (literally the AC4.3 "no external calls" mechanism — a grep-for-fetch meta-check with a seeded red fixture), `retro-index.js:isStubRetro` (line-anchored `[FILL IN]` detector).
- **New:** one vitest suite `tests/docs-hygiene.test.js` + pure lib `tools/lib/doc-hygiene.js` (each check → `{hard, soft}`); the hard-fail-vs-soft-report wiring; broadening `walkMarkdown`'s hard-coded `.planning/` root; the map-drift fold-in.
- **Hard checks:** internal `.md` link resolves to an existing **file** (anchor/slug resolution stays **soft** — GFM slug in a hard gate = false-reds); roster count in prose matches filesystem (README, CLAUDE.md, `docs/map/index.html` — AC4.1/4.5); version consistency `plugin.json`↔`marketplace.json`↔`CHANGELOG` (the B7 class); INDEX regen idempotent.
- **Soft:** anchor/slug resolution; prose staleness (stale "Last updated"/"N tests" claims); "file has no curated INDEX note yet."
- **`[FILL IN]` check — narrow allowlist, not a broad scan.** ~10 `.planning/` files contain `[FILL IN`, most legitimately (all stub retros; docs that merely *reference* the marker). Apply the hard check to a **narrow allowlist of must-be-complete published docs** (README, top-level `docs/*.md` minus templates) with the line-anchored `isStubRetro` primitive; if a clean allowlist can't be drawn, downgrade to SOFT (weakest hard-fail candidate).
- **Single roster authority:** `validate-plugin.js`'s `REQUIRED_*` are module-private + only a required-subset (list 4 scanners, not all 26 agents) → **extract `tools/lib/roster.js`** (filesystem-glob counts), imported by both the validator and the hygiene suite.
- **Reconcile-before-guard:** the 15-vs-16 drift is live; PLAN must fix every prose count (defined rule: `commands/*.md` = ground truth = **16**; re-verify 26 agents / 21 skills / test-count in CLAUDE.md too) **then** land the guard.

### FR5 — append-log eviction (evict-with-anchors) — the risky piece
- **Call: relocate verbatim behind a dated pointer; NO distilled card.** No AC needs distillation; a BYTE-identical section move satisfies AC5.1–5.5 at zero distortion risk (the degenerate case of the E1/E2 §5 "distill→verify→evict" pattern — same zero-semantic-risk shape as vector-2's whole-body relocate). So the NFR's human-eyeball semantic gate is **not needed** for FR5.
- **Eviction unit = the `## YYYY-MM-DD` date-section** (heading→next `## `). **Selector = date cutoff** at the milestone-open boundary (`2026-07-15`), strict `<` (a boundary-date collision — `M4.5.E11 PLAN` and `M5 opens` both 2026-07-15 — means `<` keeps all 07-15 sections live, the safe direction). Undatable section → stays live (conservative). Granularity = milestone (keeps M5's closed Epics E1/E2, dated 07-16, live — correct per AC5.4).
- **Reuse:** `relocateFaithful` (BYTE mode) + `conserves(BYTE)` + `createSnapshotter` + `.state.lock` + pre-apply tag + `scanDanglingLinks`/`enforceNoDangling` + TOCTOU + dry-run. **New (a 4th planner, `senseAppendLogEvict`):** date-section parser + date-cutoff classifier + pointer format + byte-stable idempotency marker + the **anchor-resolvability gate** (the reused dangling gate checks only `](path.md)` links — bare `D-…` IDs are invisible to it, so this net-new check is the only thing that realizes "with-*anchors*"). **No `verifyCardCoverage` card** (say so explicitly or the planner drags in retro-coverage machinery).
- **History placement: per-milestone `.planning/archive/M{n}/DECISIONS.md`** (reuse the archive-dir convention), NOT one unbounded `DECISIONS-HISTORY.md` (which recreates the bloat FR5 kills) — and per-milestone gives idempotent crash-reuse for free (`reuseExisting`). Reuse the *directory convention* + `relocateFaithful`, **not** `applyArchiveTree`'s mover (keys on `EPIC_PREFIXED_RE`, moves whole files).
- **Compatibility:** `checkpoint.js:appendDecisionsFile` appends dated-today `## …` sections (≥ boundary → never touched); `add.js` denylists `DECISIONS.md`. Confirmed compatible (AC5.4). Idempotency (AC5.5): pointer marker + skip-on-rescan + per-milestone reuse.
- **Hard dependency FR3→FR5** (make it explicit): 669 bare `D-…` refs resolve ONLY through FR3's index (only 2 hard file-level links, no anchors). FR5's apply regenerates the index **in-transaction**; if FR3 can't resolve 100% of evicted D-IDs → **fail-closed to detect-only**.

### FR6 — v2→v3 migrate + rollout + retire Curator
- **Reuse:** the whole stamp/banner/migrate/archive-tree machinery — `spliceDocsLayoutVersion`/`readDocsLayoutVersion`, `hooks/warn-layout-drift.js:decideLayoutDrift`, `status.js:readLayoutBanner`/`decideLayoutBanner` (all unchanged; a v2-stamped project banners the moment CURRENT=3), and the generic moveMap-driven rewrite spine (`archive-tree.js:computeLinkEdits`/`computeProseEdits`/`applyKeyedReplacements` + the mover).
- **New:** `senseV3Rename(baseDir)` emitting the **fixed** rename moveMap entry (`FUTURE-IDEAS.md → ISSUES-INBOX.md` + ledger), merged into `archiveSense.moveMap` (NOT `planArchiveMoves`, which is retro-derived); the v2→v3 plan wired into `senseProject`/`applyMigrate`; BACKLOG-from-snapshot; born-on-v3 (lazy-create + stamp `docs_layout_version: 3` at init); **the v3-conformance predicate** (H2 — the highest-risk item; land+test before bumping the version constant).
- **Bump BOTH** `migrate-memory.js:CURRENT_LAYOUT_VERSION` and `layout-stamp.js:LAYOUT_VERSION` 2→3 (the sync-assertion tests force equality).
- **Rename-scope:** **exclude `archive/`** from the rename's rewrite + residual-scan (frozen history; bare-word mentions don't dangle — only literal `.planning/FUTURE-IDEAS.md` path strings + `](…)` links do). Otherwise Signal's own dogfood (AC6.5) churns archived history or aborts.
- **Born-on-v3:** `/sig:add` **lazily creates** `ISSUES-INBOX.md` on first capture (behavior change — today `captureToDestination` throws `missingFileError`); same for BACKLOG/BUGS; `new-project.md`/`init.md` stamp `docs_layout_version: 3`.
- **Retire `ship.md §8`** Curator block; native FR3 index-regen + FR4 hygiene + FR2 sweep replace it. Update the `curator-dormant-on-signal-planning` memory + a DECISIONS entry **when FR3 lands**.
