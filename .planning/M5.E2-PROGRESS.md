## EXECUTE design pins (advisor pass, 2026-07-17 — before any code)

Two task-blocking corrections surfaced + **empirically confirmed** before writing t2/t3. Both guard the B8-catastrophe class this slice exists to prevent:

1. **`verifyFaithful` (= `verifyCardCoverage`) is NOT the vector-1 gate.** Confirmed: `verifyCardCoverage(pureProse, '')` → `pass:true` (a *total* deletion of free narrative with no IDs/dates/status-tokens waves straight through — it proves discrete-token coverage, not prose conservation). **Word/token conservation is THE vector-1 gate** (cross-cutting §2). Consequence for the strict proof-of-fail: **the B8-catastrophe fixture (S1.t3) must be pure prose with NO rescuing IDs**, so only word-conservation can catch the drop — a fixture with IDs false-greens through `verifyCardCoverage` and proves the wrong gate. `verifyFaithful` stays a *separate* ID/date/status-token backstop.
2. **t2 lock boundary decided now (not deferred to t8).** The §9 non-reentrant-lock hazard generalizes to the stamp write: a self-locking `setDocsLayoutVersion` called inside the coarse-locked apply would throw "another state write is running" (<5s) or stale-unlink the coarse `.state.lock` (>5s). **Fix:** `setDocsLayoutVersion` = a lock-free pure core (`spliceDocsLayoutVersion(text,n)→text`, raw-line splice, no I/O/lock) + a self-locking convenience wrapper for standalone use. The harness (t7/t8) holds the coarse `.state.lock` ONCE and calls the pure core — never the wrapper. Mirrors the `relocateFaithful` lock-free-primitive pattern.

## EXECUTE design pins — t7 (advisor pass #2, 2026-07-17)

Two BLOCKING corrections for the safety harness, plus two test-sharpening refinements. t7 sub-splits: **t7a** git-state probe · **t7b** apply engine (compose + lock + TOCTOU + rollback) · **t7c** dry-run 3-tier render + pre-apply dangling baseline.

3. **V1→V2 must compose in ONE apply invocation, or idempotency breaks + nextpass keeps a 529 KB body.** Vector-1 de-prose relocates frontmatter prose INTO the body → the post-V1 body can exceed the 8 KB vector-2 threshold → a naive "apply sensed vectors once" harness leaves `v2.candidate:true` → t8 won't stamp → a 2nd `--apply` fires V2 and changes files (fails t8 zero-diff). **Fix:** the apply engine chains in memory under the single coarse lock — `text=raw; if V1: text=deproseFrontmatter(text).newText; if senseState(text).v2.candidate: write STATE-HISTORY byte-identical to that body + text=planVector2(text).newText; then one atomicWrite + stamp`. Order V1-then-V2 (all narrative lands in STATE-HISTORY, not split). Discriminating test: a fixture whose de-prose pushes the body past 8 KB → one `--apply` → conformant+stamped → 2nd `--apply` byte-identical. (Carry-over open-items-swept-to-history = an S4 human-eyeball faithfulness call, not an S1 mechanical blocker.)
4. **`--force` dirty-tree rollback must be SURGICAL, never `git reset --hard`/`git checkout .`.** A blanket reset on a dirty tree discards the user's pre-existing uncommitted work — the exact data-loss class this command prevents. **Fix:** snapshot the pre-apply raw bytes of exactly the files the plan will mutate (into `.planning/.migrate/`) and rollback restores per-file from that snapshot (git AND fs paths). The git tag + printed `git reset --hard <tag>` revert line is for the CLEAN case only — never print it after `--force` on a dirty tree.
5. **TOCTOU:** hash the raw bytes of every file the plan will mutate; re-hash at apply; mismatch → abort BEFORE any write (all-or-nothing).
6. **Lock proof-of-fail (§9):** assert the apply holds `.state.lock` once, and that a self-locking wrapper (`setDocsLayoutVersion`/`applyDeproseVector1`/`relocateInlinedBody`) called UNDER that held lock throws "another state write is running".
