From PLAN § Risks:

| Risk | Mitigation in plan? |
|---|---|

| CHANGELOG/README conflict if S1+S2 separate sessions | ✓ Strict gate ensures sequencing |

**Additional risks not explicitly listed in plan, surfaced by assumptions-analyzer:**

- **Patterns not deterministic from a test harness.** S1.t1 + S1.t2 are the validation; if patterns prove non-deterministic, S1.t4's "red state" can't be reliably anchored. **Mitigation:** Layer B tests assert against the captured `actual/` fixture (which IS deterministic — it's a file on disk); Layer C tests assert against `commands/init.md` (also deterministic). The non-determinism of LIVE LLM output is exactly what the 3-layer architecture sidesteps. **Risk acknowledged + mitigated by architecture.**

- **Model version change invalidates fixture.** If Claude model upgrades between fixture-capture and E7 ship, the captured `actual/` may not reproduce. **Mitigation:** Layer B + Layer C tests don't run the model — they assert against on-disk fixtures + `commands/init.md`. The fixture is a historical artifact, not a "current behavior" snapshot. **Risk acknowledged + mitigated.**

**Verdict: PASS.** All known risks have mitigations; assumptions-analyzer red flags are addressed by the 3-layer architecture.
