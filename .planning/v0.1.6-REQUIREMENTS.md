# v0.1.6 — Doc-Integrity Guardrail — Requirements

**Shape:** lightweight patch (no Epic ID) — DISCUSS decisions locked 2026-07-13 (see `DECISIONS.md` § v0.1.6). Tracked as `current_epic: v0.1.6`; artifacts `v0.1.6-*.md`.
**Tier:** FULL / `gate_strictness: strict`. Two product-altitude gray areas gated via `AskUserQuestion`; the rest decided at product altitude.
**Test baseline:** 854 green. **No new runtime deps.** Strict Nyquist (every AC tested except documented verification-only ACs).

## Objective

Prevent *new* documentation-integrity pathology at the point it's created, and converge two capture/planning papercuts. This is a **guardrail**, not a remediation: it stops prose from landing in STATE.md frontmatter, flags an over-budget STATE.md so growth is visible, and fixes the drain + `/sig:add`-title bugs. Eviction/repair of an already-bloated STATE.md is **out of scope → Milestone 5** (root cause: `upgradeStateFile` inlining the legacy body at `state.js:177`, plus append-without-evict).

## Scope — four pieces

1. **STATE write-hook** — extend the existing `hooks/check-state-write.js` (PreToolUse Edit|Write) to block prose in frontmatter, plus a read-time size banner.
2. **Drain blockquote fix** — `tools/lib/drain.js` disposition detector recognizes `> **Promoted…**` stamps.
3. **`/sig:add` title fix** — `tools/lib/add.js` `deriveHeading` cuts at a clause boundary.
4. **BUGS.md move** — relocate 3 bug-flavored FUTURE-IDEAS entries into the existing `.planning/BUGS.md`.

## Functional Requirements

### FR1 — Block malformed frontmatter at write (extend the hook)

Extend `hooks/check-state-write.js` (today: blocks an Epic-close SHIP write when the matching retro file is absent) to **also** block a proposed STATE.md write that puts **prose** into the `completed_phases` or `blockers` frontmatter fields. Core logic lives in the tested lib extension point (`checkProposedStateWrite` in `tools/lib/retrospective.js`, or a sibling), not inline in the hook, so it's unit-testable.

- "Malformed" = violates the compact shape: `completed_phases` entries must be single-line `PHASE (YYYY-MM-DD)`-shaped (no embedded newline; bounded length); `blockers` entries must be objects `{id, text, raisedAt}` with a bounded `text`. A multi-line / narrative-blob value → **block (exit 2)** with a reason naming the offending field.
- Must **not** regress the existing retro-absence SHIP block.
- Fail-open on hook-infra conditions (no stdin / malformed JSON / non-Edit-Write tool / non-STATE path) → exit 0.

**Acceptance criteria**
- **AC1.1** — A write setting a `completed_phases` entry that contains a newline or exceeds the length budget → hook exit 2, reason names `completed_phases`.
- **AC1.2** — A write setting a `blockers[].text` that is a narrative blob far past the budget → hook exit 2.
- **AC1.3** — A well-formed STATE write (compact entries) → exit 0 (allowed).
- **AC1.4** — The existing retro-absence SHIP block still fires unchanged (no regression).
- **AC1.5** — Malformed hook input / non-STATE path / non-Edit-Write tool → exit 0 (fail-open).

### FR2 — Flag oversized STATE.md at read-time (banner, not block)

Add a read-time size check that surfaces a banner at the read surfaces that already render banners: `/sig:resume`, `/sig:status`, `/sig:checkpoint`. **Advisory only** — never blocks, never mutates. Platform-agnostic (like E10's schema-drift check, deliberately not macOS-gated).

- Threshold sits **above Signal's own legitimate file (62 KB / 330 lines)** and well below the 465 KB failure. Exact value chosen in PLAN; constraint: must NOT fire on Signal's current 62 KB STATE.md, must fire well before 465 KB. (Working assumption: ~150 KB body.)
- Banner names the size, the budget, and points at M5 eviction: *"STATE.md body is {N} KB (over the {T} KB budget) — closed-work history is accumulating; automated eviction is planned for M5. Consider trimming."*
- Detect + flag **only**. No eviction, no mutation.

**Acceptance criteria**
- **AC2.1** — A STATE.md whose body exceeds the threshold → banner appears in resume / status / checkpoint.
- **AC2.2** — Signal's own current 62 KB STATE.md → **no** banner (threshold sits above it).
- **AC2.3** — A normal/tiny STATE.md → no banner.
- **AC2.4** — The check is read-only (STATE.md mtime unchanged) and never blocks a command.

### FR3 — Drain blockquote-disposition fix (`tools/lib/drain.js`)

Extend the disposition detector so an entry stamped with a leading blockquote `> **Promoted|Deferred|Merged|Shipped|Deleted YYYY-MM-DD …**` in its body is recognized as **dispositioned** (fix **option (a)** — tolerant detector; option (b), re-stamping the convention, rejected as it would force re-writing ~40 existing entries). Result: entries promoted via the 2026-07-04 blockquote convention stop resurfacing; the ~42-candidate drain converges.

**Acceptance criteria**
- **AC3.1** — An entry whose only disposition signal is a `> **Promoted 2026-07-04 → …**` blockquote → `dispositioned: true` (excluded from `listDrainCandidates`).
- **AC3.2** — Existing heading-marker (`HEADING_DISPOSED_RE`) and `**Status:**`-line detection still classify correctly (no regression).
- **AC3.3** *(verification, not a unit invariant)* — Running the detector over the live `FUTURE-IDEAS.md` yields a materially smaller candidate set than today (the blockquote-promoted entries drop out).

### FR4 — `/sig:add` derived-title clause-boundary fix (`tools/lib/add.js`)

Change `deriveHeading` to prefer the first clause/sentence boundary (em-dash `—`, period, colon, comma) before the length cap, falling back to the current word-boundary slice + 60-char cap. Applies to all three consumers (`buildFutureIdeasEntry`, `buildOpenQuestionsEntry`, `buildMilestoneEntry`) via the shared helper.

**Acceptance criteria**
- **AC4.1** — Body "closed-work narrative must leave the live file — …" → heading "Closed-work narrative must leave the live file" (cut at the em-dash), not "Closed-work narrative must" (mid-clause).
- **AC4.2** — A body with no early punctuation → falls back to the word/length cap (simple case unchanged).
- **AC4.3** — Sentence-casing of the first char + the 60-char safety cap still apply.

### FR5 — Move 3 bug items → BUGS.md; delete from FUTURE-IDEAS

Add rows to the existing `.planning/BUGS.md` (`ID | Status | Pri | Summary` schema) for the three bug-flavored FUTURE-IDEAS entries, and delete those entries from `FUTURE-IDEAS.md` (they are defects → BUGS.md per BUGS.md's own scope rule). Update the footer counter.

- **footer-drift** — status `fixed` (shipped in v0.1.5 / E10.S3: `add.js` footer repair + lint).
- **drain-blockquote** — status `fixed` (this batch, FR3).
- **add-derived-title** — status `fixed` (this batch, FR4).
- Sequencing: catalog as `confirmed` first, flip to `fixed` as each fix lands (bookkeeping order).

**Acceptance criteria**
- **AC5.1** — BUGS.md has 3 new rows with correct status + the footer counter updated.
- **AC5.2** — The 3 entries are removed from `FUTURE-IDEAS.md`.
- **AC5.3** — No other FUTURE-IDEAS entries are disturbed; drain over the trimmed file still parses.

## Non-Functional Requirements (mirrors D-E10-10)

- **Runtime/service NFRs N/A** — no service or HTTP surface (CLI-plugin file-guard tooling). Health probe / graceful shutdown / request logging / security headers / rate limiting do not apply.
- **Fail-open** — the hook never wedges a write on its own error (matches existing hook philosophy).
- **Backward-compatible** — existing hook behavior + STATE `schema_version: 1` preserved; **no schema bump**.
- **No new runtime dependencies.**
- **Platform-agnostic** read-time banner (not macOS-gated).
- **Read-only** size check — no STATE.md mtime change.
- **Test baseline 854 stays green**; strict Nyquist — every AC has a test except documented verification-only ACs (AC3.3).

## Assumptions to validate in PLAN

1. **A PreToolUse hook cannot emit a non-blocking warning that surfaces to the user** (exit 0 = silent allow, exit 2 = block). Baseline design puts the size-warn at read-time. PLAN/research confirms; if a non-blocking warn channel exists, size-warn *may* additionally live at write-time.
2. **Size threshold** — the exact byte/line value (constraint: >62 KB body, <465 KB). Working assumption ~150 KB.
3. **Blockquote disposition variants** — PLAN scans the live `FUTURE-IDEAS.md` to confirm the stamp verbs/shape the regex must cover.

- **Drain fix option (b)** (change promotion convention to a `**Status:**` line) → not needed; (a) is backward-compatible.
- **Epic-native flow** (committed next Epic) → separate.
