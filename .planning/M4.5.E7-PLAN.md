### S1.t5 — Build `embedSection` JS helper in `tools/lib/landscape.js`

**Goal:** Pull the "verbatim embed of structure-scan table" out of LLM discretion. New deterministic helper returns the bytes the LLM is supposed to copy.

**Acceptance:**
- `tools/lib/landscape.js` exports new function `embedSection(scanContent, heading, { preserveFences = true } = {})`. Returns the substring of `scanContent` from `## {heading}\n` through (but not including) the next `## ` heading at the same level, with trailing whitespace stripped but interior content (including tables, fenced code, bullets) preserved verbatim.
- 3 new unit tests in `tests/landscape.test.js`:
  - "embedSection returns full section body verbatim for the structure-scan Source Tree table"
  - "embedSection returns null for missing heading"
  - "embedSection preserves fenced-code blocks and table cell content"
- All 3 tests fail before the helper is written (RED), pass after (GREEN).

**Verify:** `npm test -- landscape` shows 3 new tests passing; existing landscape tests stay green.

**Test strategy:** TDD red→green→commit. Write the test first; confirm it fails with `embedSection is not a function`; implement; confirm green.

**Complexity:** M (30–45 min — small helper, 3 tests, TDD cycle).

**Files touched:** `tools/lib/landscape.js`, `tests/landscape.test.js`.
