# M5.E2 — Auto-sensing migrate command — PLAN

**Phase:** PLAN complete → EXECUTE next. **Tier:** FULL / `gate_strictness: strict`. **Spec:** `M5.E2-REQUIREMENTS.md`. **Research:** `M5.E2-RESEARCH.md`. **Decisions:** D-M5E2-1…5.

## Phase goal

Ship `/sig:migrate-memory` — a safe, auto-sensing command that reorganizes the **invoking** project's `.planning/` docs to the FR1 model (de-prose frontmatter, relocate bloated bodies, evict closed-Epic narrative, build the archive tree), **relocate-never-delete**, dry-run-default, git-reversible — plus the `docs_layout_version` stamp + upgrade banner — releasable bundled with E1 as the doc-bloat fix.

## Cross-cutting constraints (apply to EVERY slice/task — from research §5/§6)

1. **Raw frontmatter lines, never `stringifyYaml` round-trip.** Splice offending list entries out of the raw text (as `checkStateFrontmatterShape` does). De-risks FM1/FM2/FM4/FM8 at once. A full re-serialize is a defect here.
2. **Move-never-delete + the B8 invariant.** Every relocation: the removed content must appear in its new home *before* the source is shortened. **"Body/frontmatter shrank but new home didn't grow" = a deletion = HARD FAIL.** Archive/verbatim moves assert **byte-identical**; in-body relocations assert **word/token conservation** (indent-agnostic).
3. **Dry-run default; apply leaves changes STAGED-but-uncommitted** (D-research §6: this runs in stranger repos — Signal doesn't auto-commit someone else's work) + prints the pre-apply tag `pre-migrate-memory-<ISO>` and the exact `git revert`/reset line. `--apply` required to write; refuses a dirty tree without `--force`.
4. **TOCTOU binding.** Apply re-reads disk; hash the inputs at dry-run and abort at apply if they changed (or collapse to one dry-run→confirm→apply invocation that never re-reads).
5. **Blocking verify, not advisory.** Any post-apply dangling link (file OR anchor), residual flat path, or conservation failure → **hard abort + rollback**, never a `console.log`-after-commit.
6. **Fail-open + cross-platform.** POSIX `/` in rewritten links; `\r?\n` tolerant; hooks/detectors cap scans at 1 MB (`FRONTMATTER_SCAN_CEILING`) and never crash a session.
7. **§5 blind spot.** Green tests prove the mechanical move + token-coverage, NEVER semantic faithfulness. The faithfulness proof is the **S4 dogfood eyeball + REVIEW** — do not mark faithfulness "done" on a passing suite.
8. **Plan artifact** (the editable dry-run plan + proposed cards) lives in a scratch/`.planning/.migrate/` working dir **removed on apply AND on abandon** (a dry-run never applied must not leave a scratch dir in a stranger's tree) — never a tracked `.planning/` path (write-guard + single-home).
9. **Lock composition (non-reentrant — plan-checker fix 1).** `withStateLock` uses `O_EXCL` and is **non-reentrant** (`file-lock.js:55` `openSync(...,'wx')` throws on a fresh same-path lock). So: hold the **coarse migrate-level lock once** for the whole apply, and do all within-STATE moves via the **lock-free inner `relocateFaithful` spine** (`evict.js:343-391`, which does not self-lock). **Never call the self-locking `evictEpicNarrative` (`evict.js:321`) under the coarse lock** — it would hard-abort, then silently unlink the coarse lock after its 5 s TTL.
10. **INDEX descope (plan-checker fix 3).** E2's migrate does **NOT** regenerate `INDEX.md` — Signal's is hand-curated (memory `curator-dormant-on-signal-planning`) and the generated-INDEX is an **E3 pre-decision**, so auto-writing it here would regress the S4.t1 dogfood target and break the E2/E3 line. The migrate leaves `INDEX.md` untouched and, if its moves left INDEX references stale/dangling, **flags "INDEX.md may need a refresh"** in the dry-run — it never rewrites it. (Supersedes the REQUIREMENTS "one-time index output" phrase.)
