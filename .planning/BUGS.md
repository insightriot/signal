# Signal ‚Äî Bugs & Findings

Lightweight catalog of bugs, defects, and verified findings for Signal itself.
Markdown-native by design ‚Äî GitHub Issues are deferred until Signal has live
users (see `FUTURE-IDEAS.md` ¬ß "GitHub Issues for work-item tracking ‚Äî deferred").

**Discipline** (per the "always catalog bugs" norm): catalog a finding the moment
it surfaces ‚Üí **triage** (read the code / confirm the root cause) ‚Üí mark
**confirmed** or **dismissed**. Never graduate a finding to a confirmed bug
unverified.

- **Status:** `needs-triage` ¬∑ `confirmed` ¬∑ `dismissed` ¬∑ `fixed`
- **Priority:** `P1` ship-blocker ¬∑ `P2` should-fix ¬∑ `P3` nice-to-have
- **Scope:** defects + verified findings only. Architectural ideas ‚Üí `FUTURE-IDEAS.md`;
  design questions ‚Üí `OPEN-QUESTIONS.md`; decisions ‚Üí `DECISIONS.md`.
