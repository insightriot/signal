## 2. Important Issues (resolved in-phase per PASS-WITH-FIXES)

### I-1. `audit-network-calls.js` exit-code 2 undocumented (contract drift)

- **File:** `tools/audit-network-calls.js:3-29` (header docstring).
- **Issue:** The script header documented only exit 0 ("no patterns matched") and exit 1 ("at least one pattern matched"). The implementation also exits 2 in two error paths (`main()` at `tools/audit-network-calls.js:108` and `:113`) when the explicit directory argument doesn't exist or isn't a directory. Callers relying on the documented contract — e.g., CI gating that branches on `0 / non-0` — could be misled by an undocumented third exit code.
- **Why important, not nit:** the script's whole purpose is to provide a verifiable privacy claim. The exit-code contract is the API third parties read. Doc drift on the API is a real defect for an "audit you can trust" tool.
- **Fix landed in-phase:** commit `4edc0a1`. Added `2 — usage error (...)` line to the EXIT block; behavior unchanged.

## 3. Suggestions (optional improvements; one applied, others deferred)

### S-1. Remove unused `basename` import — APPLIED in-phase

- **File:** `tools/audit-network-calls.js:32` (pre-fix).
- **Issue:** `basename` imported from `node:path` but never called.
- **Fix landed in-phase:** commit `4edc0a1`. Removed from import list.

### S-2. `extractFact()` regex fragility — DEFERRED

- **File:** `tests/cross-file-consistency.test.js:20-24`.
- **Issue:** `extractFact(label)` builds a regex from a string label. Callers must pre-escape regex metacharacters (e.g., `extractFact('Node\\.js')` is correct; `extractFact('Node.js')` would silently treat `.` as a wildcard). Works as long as all 4 call sites stay disciplined; would bite a future maintainer adding a 5th label without noticing.
- **Why deferred, not in-phase:** abstract-on-first-use principle. Four call sites is below the threshold where a helper earns its keep. Suggest re-evaluating when this file gets its 5th call to `extractFact`.

### S-3. `relative(ROOT, hit.file)` produces awkward output for explicit-arg invocations — DEFERRED

- **File:** `tools/audit-network-calls.js:123`.
- **Issue:** When the script is invoked with an explicit directory argument outside the repo root (e.g., a `/tmp/...` fixture from the test suite), `relative(ROOT, hit.file)` produces a path with leading `../../...` segments. Output reads awkwardly: `✗ found network call: ../../tmp/foo/with-fetch.js:5`.
- **Why deferred, not in-phase:** the seeded-fail test (`tests/audit-network-calls.test.js`) only matches `/with-fetch\.js/`, which works regardless of the path prefix. The awkward output only appears when humans pass external paths — a power-user path, not a default. Fix would conditionally use `relative()` for in-repo paths and the absolute path otherwise. ~10 LOC of branching for minor cosmetic improvement. Defer until a real user complains.

### S-4. Recursive `walk()` has no symlink-loop protection — DEFERRED

- **File:** `tools/audit-network-calls.js:57-75`.
- **Issue:** If a symlink in any scanned directory points to an ancestor, `walk()` recurses infinitely until stack overflow.
- **Why deferred, not in-phase:** Signal's repo has no symlinks in scope. Adding `realpath`-based loop detection is a real complexity bump for a hypothetical threat. Defer until a user reports a runaway recursion (very unlikely on a docs-and-tools repo). If it ever ships, would also use a `Set` of visited inodes.
