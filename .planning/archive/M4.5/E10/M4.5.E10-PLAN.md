## Architecture decisions (locked this phase)

Three PLAN-gate scope decisions (user-gated 2026-07-05) + the plumbing forks resolved from research:

- **AD1 (SD1) — FR1 kept, scoped honestly.** The resolver serves Epic-prefixed/hand-managed projects (Signal-on-Signal). `current_epic` is never written by a command and phase artifacts are phase-prefixed, so pattern-0 only fires where both are hand-managed. No `current_epic` writer, no Epic-prefixed artifact writing (that's barred M5-scale surface). REQUIREMENTS/README framing must say "Epic-prefixed projects," not "strangers."
- **AD2 (SD2) — schema-drift lives in `/sig:status` + `/sig:resume`, not `/sig:doctor`.** `/sig:doctor` is macOS-gated + `~/.claude`-scoped; putting it there hides the warning from Linux/WSL testers and breaks E8's charter. The pure detector lives in `tools/lib/state.js` (co-located with `SCHEMA_VERSION` + `parseFrontmatter`); the banner surfaces where E6/E8 project-state banners already live. `commands/doctor.md` is untouched. `MILESTONE-4.5.md` § E10 bullet 5 updated to match (docs-accuracy).
- **AD3 (SD3) — privacy docs reconciled.** README §Privacy + `docs/launch-post.md` amended to state the two real network calls (GitHub version check; read-only `git fetch` to the user's own remote — no data sent). No `audit-network-calls.js` hardening this Epic.
- **AD4 — `SCHEMA_VERSION` exported** from `state.js` (prereq for AD2's detector; currently module-private).
- **AD5 — guards split pure/announce.** `detect+recover` in the pure helpers (`{content, recovered, kind}`); announcement in the command/caller layer. `parseEntries`/`insertAboveFooter`/`rewriteFooter` keep their bare-return signatures (prefer-addition; snapshot tests pin them).
- **AD6 — FR5 detector bypasses `readState`** (it throws on both ahead + missing-key, indistinguishably); uses `parseFrontmatter` + numeric compare so AC5.3 "rather than crashing" holds.
- **AD7 — origin-drift fetch is hardened + bounded:** `GIT_TERMINAL_PROMPT=0`, neutralized askpass/credential helpers, `SSH BatchMode`, `execFileSync timeout` + `SIGKILL`. Every failure → `{stale:false}`, never throws. Fetch writes `.git/` not `.planning/`, so `/sig:status`'s read-only gate holds (state it, pre-empt REVIEW).
- **AD8 — resolver has one consumer** (`/sig:resume`); extracted for testability only, not threaded into status/checkpoint (they don't read phase artifacts).
- **AD9 — FR6 harness is cwd-driven**: spawn the hook with `cwd` = planted fixture; the stdin payload is decorative (the hook ignores it). No stdin parsing added. The real-runtime handshake is the essential manual leg (AC6.4).

## Risks & mitigations
| Risk | Impact | Mitigation |
|---|---|---|
| Origin-drift fetch hangs on auth-prompt remote | High (blocks a command) | AD7 hardened env (`GIT_TERMINAL_PROMPT=0`, neutralized askpass, SSH BatchMode) + `execFileSync timeout`+SIGKILL; fail-open catch |
| `/sig:status` latency from fetch | Med | ~2s timeout, skip-on-timeout; `ls-remote` sha pre-check optional; document `.git/`-not-`.planning/` so read-only gate holds |
| `resolveArtifactPath` path traversal via `current_epic` | Med (file escape) | sanitize `/^[A-Za-z0-9._-]+$/` + path-confine; traversal test |
| `markFresh` throws `StateSchemaError` (not just write/lock) | Med | catch-all wrapper in discuss/plan |
| FR5 detector crashes on ahead/missing schema | Med | bypass `readState`; `parseFrontmatter`+numeric compare; try/catch |
| Guard "recover" corrupts a well-formed file | Med | idempotence tests (byte-identical except date/no-op); pure detect+recover, command-layer announce |
| FR6 harness over-claims coverage | Low | scope test to I/O contract; manual leg flagged essential in hooks-api.md |
