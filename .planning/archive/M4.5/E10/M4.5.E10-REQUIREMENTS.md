## Objective

Make `/sig:resume` (and its siblings `/sig:status` / `/sig:checkpoint`) brief against *current* reality, and make the `/sig:add → FUTURE-IDEAS → /sig:plan drain` capture pipe impossible to silently corrupt — so a stranger running Signal on their own project on a v0.1.x cadence never re-plans shipped work, never loses a captured idea, and never hits a mid-project schema surprise unannounced.

E10 is a release-hardening Epic in the M4.5 charter. Five of its six items are pure code/doc fixes an EXECUTE agent can complete end-to-end; the sixth (the SessionStart hook smoke test) has one leg only a live Claude Code session can run, handled via the E5 asset/human split.

## Functional Requirements & Acceptance Criteria

### FR1 — `/sig:resume` Epic-prefix artifact resolution
The resume briefing must find Epic-prefixed phase artifacts (`M4.5.E10-PLAN.md`, etc.), the actual Signal convention since the Milestone/Epic vocabulary lock.

- **AC1.1** — Given `state.current_epic = M4.5.E99` and `.planning/M4.5.E99-PLAN.md` on disk, the resolver returns that path for the PLAN artifact.
- **AC1.2** — Given a legacy project with `1-PLAN.md` and no `current_epic`, the resolver still returns `1-PLAN.md` (pattern-0 is skipped when `current_epic` is null; legacy patterns unchanged).
- **AC1.3** — Pattern precedence is 0 → 1 → 2 → 3; the Epic-prefixed form wins when both it and a legacy form exist.
- **AC1.4** — When no pattern matches, the resolver signals "not found" so the command emits its existing one-line note (no crash, no false match).
- **AC1.5** — `commands/resume.md` Step 3 prose lists pattern 0 as the first attempt.

### FR2 — Origin-drift detection
`/sig:resume`, `/sig:status`, `/sig:checkpoint` warn when origin has commits the local `STATE.md` doesn't reflect.

- **AC2.1** — `isStaleVsOrigin` returns `{stale:true, aheadCount:N, commits, touchedPlanning}` when `origin/<default>` is N commits ahead of `last_updated_commit`.
- **AC2.2** — Returns `{stale:false}` when local `STATE.md.last_updated_commit` equals `origin/<default>` HEAD.
- **AC2.3** — `touchedPlanning` is true iff at least one unpulled commit touched a `.planning/` path.
- **AC2.4** — Offline / no `origin` remote / not-a-git-repo → no-op `{stale:false}` (+ optional one-line stderr note); the command completes normally.
- **AC2.5** — Default branch resolved via `origin/HEAD`, falling back to `main` when `origin/HEAD` is unset.
- **AC2.6** — All three commands render a non-blocking banner when `stale:true`, highlighting `.planning/`-touching commits; the briefing/report continues.
- **AC2.7** — The fetch is bounded by a short timeout; on timeout the drift-check is skipped for that invocation and the command still returns promptly.

### FR3 — STATE auto-update Option A (discuss + plan)
DISCUSS and PLAN refresh `STATE.md` frontmatter at end-of-workflow, closing the gap where between-EXECUTE work left `last_updated_commit` stale.

- **AC3.1** — `commands/discuss.md` ends its workflow with a `markFresh(baseDir, {commit})` step, same shape as verify/review/ship.
- **AC3.2** — `commands/plan.md` ends its workflow with the same step.
- **AC3.3** — Both steps degrade gracefully on `markFresh` failure (lock contention / git unavailable) — a warning, not a phase-gate abort — matching the existing verify/review/ship handling.
- **AC3.4** — After a DISCUSS or PLAN commit followed by the refresh, `/sig:resume`'s local staleness banner reads fresh.

### FR4 — Capture-pipe guards
Neither a dangling code fence nor a stranded footer can silently swallow ideas.

- **AC4.1** — `parseEntries` on content with an unclosed fence at EOF still returns the entries that appear below the dangling fence (they are not hidden).
- **AC4.2** — That parse emits a warning identifying the dangling-fence condition.
- **AC4.3** — A well-formed file (balanced fences) parses identically to today (no behavior change / no spurious warning).
- **AC4.4** — `/sig:add` into a `FUTURE-IDEAS.md` with content stranded below the footer repairs the footer to true EOF and lands the new entry correctly (above the now-EOF footer), and announces the repair.
- **AC4.5** — A well-formed `FUTURE-IDEAS.md` (footer already at EOF) is written exactly as today (no spurious repair).
- **AC4.6** — A validator/lint check fails when `FUTURE-IDEAS.md` has non-whitespace content after its `*Last updated:*` footer.

### FR5 — `/sig:doctor` upgrade-path (schema-drift) diagnostic
`/sig:doctor` detects when an in-flight project's STATE schema is behind the installed plugin's expectation.

- **AC5.1** — Given a project `STATE.md` with `schema_version` less than the plugin's `SCHEMA_VERSION`, `/sig:doctor` reports schema drift and points to the auto-migration path.
- **AC5.2** — Given `schema_version` equal to `SCHEMA_VERSION`, no schema-drift finding.
- **AC5.3** — Given `schema_version` greater than `SCHEMA_VERSION` (plugin too old), `/sig:doctor` reports the fail-closed condition rather than crashing.
- **AC5.4** — A project with no `.planning/STATE.md` (install-only context) yields no schema-drift finding (the detector is scoped to in-flight projects).
- **AC5.5** — The diagnostic is read-only (no mutation of the project's STATE.md).

### FR6 — SessionStart hook smoke test + `references/hooks-api.md`
The E9 SessionStart(resume) handshake is verified and the hook contract is documented.

- **AC6.1** — An automated test spawns `hooks/warn-dirty-execute.js` with a synthesized `SessionStart(resume)` stdin payload for a dirty-EXECUTE state and asserts the stdout JSON matches `{hookSpecificOutput: {hookEventName: "SessionStart", additionalContext: <non-empty>}}` with exit 0.
- **AC6.2** — The same harness asserts a clean (non-dirty) state produces no `additionalContext` injection.
- **AC6.3** — `references/hooks-api.md` documents each hook event Signal uses (`PreToolUse(Edit|Write)`, `SessionStart(resume|startup|clear)`) with its stdin shape, stdout/exit-code contract, `hooks.json` matcher wiring, and Signal's fail-open convention.
- **AC6.4** — `references/hooks-api.md` includes the manual real-session smoke procedure (synthesize dirty state → quit → reopen/`claude --continue` → observe injected warning → restore) as the human-run attestation for the true end-to-end leg.

## Non-Functional Requirements

Per D-E10-10, the FULL runtime ops checklist is **N/A** (no service, no HTTP surface): health/liveness probe — N/A; graceful shutdown/signal handling — N/A; security headers (CSP/HSTS/X-Content-Type-Options) — N/A; rate limiting — N/A; structured request logging — N/A. The binding NFRs for this Epic:

- **NFR1 — Fail-open resilience.** Every new git/network/filesystem call (fetch, `rev-parse`, schema read, guard checks) degrades to a no-op with at most a one-line stderr note; never blocks, crashes, or aborts a command. This is the load-bearing NFR — the whole Epic is trust-hardening, so a hardening check that itself breaks a command is a net loss.
- **NFR2 — Performance-bounded network.** The origin-drift `git fetch` is read-only and timeout-bounded so `/sig:status`'s fast one-screen contract holds; on timeout, drift-check is skipped, not retried.
- **NFR3 — Backwards-compat.** No `schema_version` bump. Legacy numeric-prefix artifacts, projects without an `origin` remote, and non-git directories all keep working. The resolver's pattern-0 only fires when `current_epic` is present.
- **NFR4 — Transparency.** Guards announce any file normalization (footer repair) and any recovered-from-malformation parse (dangling fence); never a silent rewrite of a git-tracked file.
- **NFR5 — No new runtime dependencies.** Pure `node:` built-ins + existing helpers (`atomicWrite`, `execFileSync` injection pattern).
- **NFR6 — Test parity + strict Nyquist.** 777-test baseline stays green; every AC above maps to a test, except AC6.4's real-session leg, which maps to a documented human attestation.
