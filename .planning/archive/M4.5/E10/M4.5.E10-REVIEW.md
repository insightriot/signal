## Important Issues (all fixed in-phase)

### F1 — `isStaleVsOrigin` (and `isStateStale`) crashed on a schema-drifted STATE.md — *flagged by BOTH agents*
`tools/lib/state.js`. `readState` throws `StateSchemaError` on an ahead / missing-key / malformed `schema_version` — exactly the files E10's schema-drift banner exists to surface. The `readState` call sat **outside** the fail-open try/catch, and the docstring + three command docs promise "never throws," so `/sig:status` (and, via `isStateStale` which runs first, `/sig:resume` + `/sig:checkpoint`) would crash on a drifted STATE.md **instead of rendering the graceful banner** — the precise irony the review caught. Security agent reproduced it empirically (`schema_version: 2` → `StateSchemaError`).
**Fix:** wrapped `readState` in try/catch → fail open in both functions; added the schema-drift case to the docstring's enumerated fail-open modes. **Verified end-to-end:** an ahead-schema STATE.md now returns `{stale:false}` from both AND the schema-drift banner renders (`⚠ STATE.md schema drift (ahead)…`). Locked by `is-stale-vs-origin.test.js` + `is-state-stale.test.js` (REVIEW F1 cases).

### F2 — drain-recovered entries had a `range` but no valid disposition index — *code-reviewer*
`tools/lib/drain.js` + `commands/plan.md`. `applyDisposition`/`applyDispositions` take an `entryIndex` into `parseEntries(content)`, but dangling-fence-recovered entries are (by construction) absent from that parse. So dispositioning a recovered entry — or a "defer all remaining" batch that swept one up — could throw `no entry at index N` or mis-stamp a different entry.
**Fix:** `listDrainCandidatesWithRecovery` now tags recovered entries `recovered: true`; `plan.md` §1b renders them for triage-awareness but excludes them from disposition + the batch (the correct action is "fix the fence, re-run"). Locked by `drain.test.js` (REVIEW F2 case).

## Suggestions (fixed in-phase — cheap, security Epic)

### Sec-2 — no option-injection guard on the user-editable `last_updated_commit` — *security-auditor*
`tools/lib/state.js`. `last_updated_commit` (user-editable YAML) is glued into a git revision range (`${stored}..${tracking}`). A crafted `--output=/path/VICTIM` produces the token `--output=/path/VICTIM..origin/main`; `git log --output=` writes an arbitrary file. **Not exploitable as composed today** (the `rev-list` gate rejects `--output` before the `log` step runs — auditor verified `{stale:false}`, no file written), but it's one refactor from Critical.
**Fix:** validate the token against `COMMIT_TOKEN_RE` (`/^[0-9A-Za-z][0-9A-Za-z._/-]*$/` — must start alphanumeric, ref-safe chars) in both functions; reject → fail open. Root-cause fix at the input boundary rather than git-arg juggling. **Verified:** a crafted `--output=` STATE.md fails open and writes **no** file. Locked by tests in both stale test files.

### F3 — `readSchemaDrift`'s `readFile` was outside the crash-safe try — *code-reviewer*
`tools/lib/state.js`. A delete-after-`existsSync` race or permission error would throw out of the function whose whole job is graceful degradation.
**Fix:** folded `readFile` into the try → degrades to `{status:'unreadable'}`.
