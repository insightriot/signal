## 2. GitHub API details (resolves OQ2)

**Endpoint:** `https://api.github.com/repos/InsightRiot/signal/tags` (NOT `/releases/latest`).

**Response (HTTP 200, ~200 bytes):**
```json
[
  {"name": "v0.1.2", "commit": {"sha": "...", "url": "..."}, "node_id": "...", ...},
  {"name": "v0.1.1", ...},
  {"name": "v0.1.0", ...}
]
```

First array element is the most recent tag by default sort order. Field is `name`, not `tag_name`.

**Rate limits:** 60 req/hr unauth per IP (verified via `x-ratelimit-limit: 60` response header). 24h cache makes the practical ceiling 1 req/day; even an unlucky CI matrix with shared IP has 60× headroom.

**Implementation choice:** native `fetch` + `AbortSignal.timeout(5000)`. Node 22+ has stable built-in fetch (no `node-fetch`/`undici` dep needed). `fetch` doesn't throw on HTTP errors — must check `response.ok` explicitly. Handle 404 / empty-array / timeout / corrupt-JSON / offline all as "no latest version known → print briefing without staleness banner."

```js
async function fetchLatestTag({ fetchFn = fetch } = {}) {
  try {
    const res = await fetchFn(
      'https://api.github.com/repos/InsightRiot/signal/tags',
      { signal: AbortSignal.timeout(5000) }
    );
    if (!res.ok) return null;
    const tags = await res.json();
    if (!Array.isArray(tags) || tags.length === 0) return null;
    return tags[0].name; // e.g. "v0.1.2"
  } catch {
    return null;
  }
}
```

**Note re v0.1.3:** Currently unreleased. `tag_name` of installed `plugin.json` (`0.1.3`) will compare as *newer than latest tag* (`v0.1.2`) — version-comparator must handle "installed > latest" without printing a staleness banner.

## 3. Detection algorithm — per-P-state sketches

All detectors take state-as-data and return `{ detected: bool, evidence: object|string[], recommendation: '--fix'|'--reinstall'|'info-only', code: 'P1'|'P2'|'P3'|'P4'|'P5' }`.

Aggregate function `runAllDetectors({homeDir})` returns `{ healthy: bool, findings: Finding[], aggregate_recommendation: '--fix'|'--reinstall'|null }`.

**Aggregate-recommendation rules** (highest severity wins):
- Any P1 OR (P2 AND P3) OR (P1 AND anything) → `--reinstall`
- P2 alone OR P3 alone OR P4 alone OR P2+P4 OR P3+P4 → `--fix`
- Only P5 → `null` (informational; doesn't change `healthy: true`)
- No findings → `null`, `healthy: true`

### detectP1_staleGitCommitSha(manifest, fsImpl)

```js
const entry = manifest.plugins?.['sig@signal']?.[0];
if (!entry) return { detected: false, code: 'P1' };
const cacheDir = entry.installPath;
if (!fsImpl.existsSync(cacheDir)) {
  return { detected: true, code: 'P1', evidence: 'cache dir missing entirely', recommendation: '--reinstall' };
}
const pluginJsonPath = path.join(cacheDir, '.claude-plugin/plugin.json');
const cachedPluginJson = JSON.parse(fsImpl.readFileSync(pluginJsonPath, 'utf8'));
if (cachedPluginJson.version !== entry.version) {
  return {
    detected: true, code: 'P1',
    evidence: { manifestVer: entry.version, cachedVer: cachedPluginJson.version, sha: entry.gitCommitSha },
    recommendation: '--reinstall',
  };
}
return { detected: false, code: 'P1' };
```

### detectP2_orphanCacheEntry(manifest, fsImpl, homeDir)

Walk `<homeDir>/.claude/plugins/cache/signal/sig/*` and `<homeDir>/.claude/plugins/cache/signal/signal/*` (covers P4 territory too — P4 detector dedupes). Collect version dirs not referenced by any `installed_plugins.json` entry.

```js
const manifestPaths = new Set(
  Object.values(manifest.plugins || {}).flat().map(e => e.installPath)
);
const orphans = [];
for (const verDir of glob('<homeDir>/.claude/plugins/cache/signal/sig/*/')) {
  if (!manifestPaths.has(verDir)) orphans.push(verDir);
}
return orphans.length
  ? { detected: true, code: 'P2', evidence: orphans, recommendation: '--fix' }
  : { detected: false, code: 'P2' };
```

### detectP3_orphanEnabledFlag(settings, manifest)

```js
const enabled = settings.enabledPlugins || {};
const installed = manifest.plugins || {};
const orphans = [];
for (const key of Object.keys(enabled)) {
  if ((key.startsWith('sig@') || key.startsWith('signal@')) && !installed[key]) {
    orphans.push(key);
  }
}
return orphans.length
  ? { detected: true, code: 'P3', evidence: orphans, recommendation: '--fix' }
  : { detected: false, code: 'P3' };
```

### detectP4_preRenameSlug(manifest, settings, fsImpl, homeDir)

```js
const hits = [];
if (fsImpl.existsSync(path.join(homeDir, '.claude/plugins/cache/signal/signal'))) {
  hits.push('cache:~/.claude/plugins/cache/signal/signal/');
}
if (manifest.plugins?.['signal@signal']) hits.push('manifest:installed_plugins.json signal@signal entry');
if (settings.enabledPlugins?.['signal@signal'] !== undefined) hits.push('settings:enabledPlugins.signal@signal');
return hits.length
  ? { detected: true, code: 'P4', evidence: hits, recommendation: '--fix' }
  : { detected: false, code: 'P4' };
```

### detectP5_sshMultiIdentity(fsImpl, homeDir)

```js
const sshConfigPath = path.join(homeDir, '.ssh/config');
if (!fsImpl.existsSync(sshConfigPath)) return { detected: false, code: 'P5' };
const content = fsImpl.readFileSync(sshConfigPath, 'utf8');
const hasMultiHost = /^Host\s+github\.com-/m.test(content);
const hasDefaultHost = /^Host\s+github\.com\s*$/m.test(content);
if (hasMultiHost && !hasDefaultHost) {
  return {
    detected: true, code: 'P5',
    evidence: 'multi-identity Host github.com-* without default Host github.com',
    recommendation: 'info-only', // P5 doesn't change healthy
  };
}
return { detected: false, code: 'P5' };
```

## 4. Implementation choices locked

### Module shape — `tools/lib/doctor.js`
- Pure detector functions (testable in isolation, no IO).
- IO-bound orchestrators: `readInstallState({homeDir, fsImpl?})`, `runDoctor({homeDir, fetchFn?})`.
- Pure script-gen functions: `buildFixScript(findings, opts)`, `buildReinstallScript(opts)`.
- IO-bound writer: `writeDoctorScript(scriptPath, content)` (delegates to `atomicWrite()`).
- Error classes: `class DoctorEnvironmentError extends Error` (non-macOS / non-`~/.claude` host), `class DoctorDetectionError` (unreadable settings.json with friendly "mid-write?" message).

### HTTP fetch
- Native `fetch` + `AbortSignal.timeout(5000)`.
- All failures collapse to `null` → no staleness banner; briefing prints normally.

### Cache
- `~/.claude/.sig-version-cache.json` — `{ fetched_at: ISO8601, data: { name: "v0.1.2" } }`.
- Reuse `tools/lib/atomic-write.js` for writes.
- Three states: miss / stale / invalid. Invalid (parse-fail, missing keys) treated as miss.
- TTL = 24*60*60*1000 ms.

### Generated script
- **Shebang:** `#!/usr/bin/env bash` (NOT `/bin/bash` — macOS ships bash 3.2; users with Homebrew bash 5 expect env-style).
- **Strictness:** `set -u -o pipefail` (NOT `set -e` — interacts badly with declined `[y/N]` prompts, which must continue rather than abort).
- **JSON edits:** `node -e "..."` inline (Signal already requires Node 22+, so `node` is guaranteed; `jq` is NOT — verified `which jq` empty on Mac Studio).
- **Pattern per step:**
  ```bash
  read -p "Execute: rm -rf /Users/.../plugins/cache/signal/sig/0.1.0/ ? [y/N] " ans
  if [[ "$ans" == "y" ]]; then
    rm -rf /Users/.../plugins/cache/signal/sig/0.1.0/
    echo "  [done]"
  else
    echo "  [skipped]"
  fi
  ```
- **Path literals:** resolved at script-generation time (not glob/wildcard) — single source of truth, hardcoded after resolution. If detection finds case-mismatched siblings (CI macOS), abort with hard error.
- **Final step is a printed instruction**, not a command:
  ```
  echo ""
  echo "Filesystem cleanup complete. Final steps require Claude Code:"
  echo "  1. Inside Claude Code, run: /reload-plugins"
  echo "  2. Then: /sig:doctor   (to verify)"
  ```
- **Min Claude Code version probe** at top of script:
  ```bash
  if ! command -v claude >/dev/null 2>&1; then
    echo "ERROR: 'claude' CLI not on PATH. Open Claude Code, ensure binary linked."
    exit 1
  fi
  # min version check elided for brevity — see S2 plan
  ```

### OS detection
- Positive allowlist: `if (process.platform !== 'darwin') { printStub(); return 0; }`.
- Extra probe: `os.homedir().startsWith('/Users/') && fs.existsSync(<homedir>/.claude)` — catches WSL-with-Linux-paths and Linux-with-macOS-paths edges.
- Stub message includes detected platform for self-diagnosis: `Detected platform: linux. /sig:doctor currently supports macOS only.`

### Exit code semantics
- `0` — healthy, no findings, no errors.
- `1` — P-state(s) detected (broken install — actionable by user).
- `2` — doctor encountered an error (parse failure, unexpected state, broken settings.json) — install state **unknown**; user should investigate manually.

CI consumers should gate on `!= 0`; dashboards can distinguish "broken install" from "broken doctor."

### Version comparison
- Hand-roll, NOT new `semver` dep. Signal currently has only `yaml` as a runtime dep; adding `semver` doubles the surface for a 4-input function.
- 3-part numeric compare. Strip leading `v`. Ignore pre-release suffixes (treat `0.1.3-rc1` as equal to `0.1.3` for staleness purposes).
- Unit test table:

| installed | latest | expected |
|---|---|---|
| `0.1.2` | `v0.1.10` | stale (1.10 > 1.2 numerically) |
| `0.1.2` | `v0.1.2` | current |
| `0.1.10` | `v0.1.2` | newer (silent; no banner) |
| `v0.1.2` | `v0.1.2` | current |
| `0.1.3-rc1` | `v0.1.3` | current (rc suffix ignored) |
| `0.1.3` | `v0.1.2` | newer (installed ahead — v0.1.3 unreleased case) |

## 6. Doctor command prior art

Borrowed conventions:

1. **Exit-1-on-findings** (from `brew doctor`) — already locked in FR2. CI-friendly.
2. **`[✓] / [!] / [X]` glyphs per finding + trailing summary** (from `flutter doctor`) — cleaner than prose.
3. **Healthy line:** `Signal v0.1.3 installed and healthy — no action needed.` (matches FR2 / AC #1).
4. **stdout for findings, stderr only for doctor-itself-crashed.** Reasonable departure from brew's stderr-for-all approach; keeps `/sig:doctor | tee` use cases simple.
5. **`--fix` semantic** — Signal's `--fix` generates-script-for-review departs from eslint/prettier's auto-apply convention. **Mitigate via help text:** `--fix` description must clearly say "writes a script you review and run, does not mutate state directly." Avoids surprise.

`gh doctor` doesn't exist (verified). No convention to borrow from there.

## 7. Risks surfaced — Top 3 for PLAN's anti-rationalization table

### Risk 1 — Test isolation (HIGHEST blast radius)

**Implicit assumption:** "test fixtures faithfully mirror `~/.claude/` and tests can mutate fixtures without touching the developer's real `~/.claude/`."

**Failure mode:** AC #3 requires "fixture filesystem comparison" post-script-execution. If script's `~/.claude/...` paths are hardcoded, running tests could execute destructive script content against the developer's real `~/.claude/`.

**Mitigation:**
- All detectors take `homeDir` as injected parameter (NOT `os.homedir()` direct).
- Script generation **templates the base path**, not hardcodes `~/.claude/` — `homeDir` is resolved at script-gen time to the absolute path from detection.
- Meta-test asserts the generated script never contains a literal `~/.claude/` (only the resolved absolute path).
- Tests pass `homeDir: <fixture-tmpdir>`; production calls pass `homeDir: os.homedir()` from the command file.

**Anti-rationalization:** "We'll just be careful with the test paths" — no, encode the isolation in the code, not the developer's vigilance.

### Risk 2 — Signal-scoped detectors (HIGH blast radius)

**Implicit assumption:** "`~/.claude/` is Signal-only territory."

**Failure mode:** Every detector that scans broad state files (`settings.json`, `installed_plugins.json`, cache root) can fire on non-Signal plugin entries. P1 on a non-Signal plugin would propose `rm -rf` against another plugin's cache.

**Mitigation:** Every detector filters by `key.startsWith('sig@') || key.startsWith('signal@')` or path-pattern `signal/sig/` / `signal/signal/` BEFORE emitting a finding. Add a fixture where a non-Signal plugin sits in `installed_plugins.json` with simulated staleness; doctor must report healthy.

**Anti-rationalization:** "The cache directory is plugin-scoped so it doesn't matter" — `settings.json` and `installed_plugins.json` are NOT plugin-scoped.

### Risk 3 — GitHub API as advisory, not authoritative

**Implicit assumption:** "`api.github.com` is reachable and well-formed."

**Failure mode:** offline / rate-limited / 404 / malformed JSON / corrupt cache. Today, `/releases/latest` 404s; the version-check must not hard-fail.

**Mitigation:**
- `/sig:status` wraps version-check in try/catch; on any failure, prints briefing normally without staleness banner. Optional one-line `(version check skipped: <reason>)` footer.
- `/sig:doctor` P-state detection does NOT depend on the API — version-check feeds only the FR6 recommendation column.
- All AC tests for FR6 inject `fetchFn` mock; tests NEVER hit live network.

**Anti-rationalization:** "We'll deal with network failure later" — the network failure case IS the modal case during the v0.1.3 unreleased window.

## 8. Additional risks surfaced (lower priority — handle in implementation but not headline)

- **Script destructive-action safety (FR4/FR5):** atomic JSON edits via `node -e` (not `sed`), `rm -rf` with resolved literal paths (not globs), no per-step dependency assumptions when prompts decline. Detection resolves a single canonical marketplace casing; abort hard on case-mismatched siblings.

- **Version comparison correctness:** hand-roll 3-part numeric compare with `v`-strip and pre-release-suffix-ignore. Test table in § 4.
- **Dogfood AC #13 substrate:** Mac Studio has live P2 + P4 right now — doctor can be exercised here. Reframe AC #13 from "biz machine specifically" to "any second macOS machine"; biz unavailability shouldn't block SHIP.
