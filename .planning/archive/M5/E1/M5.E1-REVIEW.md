## Important Issues — all 4 FIXED in-phase (PASS-WITH-FIXES)

Notably, all four cluster on the `evictEpicNarrative` / drain paths VERIFY already flagged as never-live-fired — so they're fixed **before** that path's first real Epic-close SHIP.

### I-1 — coverage gate used substring `includes` → a lossy card PASSED — **FIXED**
`tools/lib/evict.js` `verifyCardCoverage`/`isCovered`. `cardText.includes(n)` counted source `M5.E1` as covered by a card that only mentions `M5.E10` (also `FR2`↔`FR2b`, `AC1`↔`AC10`) — a false-negative in the **wrong** direction for a gate whose job is to FAIL loss. **Reproduced:** `verifyCardCoverage("Closed M5.E1. AC1. FR2.", "…M5.E10; AC10… FR2b…")` returned `pass:true`. **Fix:** bounded match `(?:^|[^A-Za-z0-9])<id>(?:[^A-Za-z0-9]|$)` (alphanumeric neighbours = "part of a longer ID"; `.`/`-` are boundaries so a trailing period / the id's own separators don't block a legit match). Post-fix the same input → `pass:false`, `missing:[M5.E1,AC1,FR2]`; an exact mention still passes. Regression tests added. ~7 LOC.

### I-2 — evicted pointer orphaned under the *previous* Epic; `## Closed work` never populated — **FIXED**
`tools/lib/evict.js` `evictEpicNarrative`. The pointer was spliced in-situ, landing under the prior `## <Epic>` heading (reads as if it belongs to that Epic), while `state-schema.md`'s skeleton advertises `## Closed work` as the pointer home — which the mechanism never wrote to. **Fix:** route the pointer under `## Closed work` when the body has that section (fallback to in-situ for a real migrated body with no skeleton); collapse the removed-section seam. Regression test asserts `pointerIdx > "## Closed work" idx`. ~15 LOC.

### I-3 — `evictEpicNarrative` wrote STATE.md **outside `withStateLock`** — **FIXED**
`tools/lib/evict.js` (read→write) — the lone STATE.md mutator skipping the 5s-TTL lock every other mutator (`transitionPhase`, `setCurrentTask`, `markFresh`, …) uses; `withStateLock` was module-private so evict *couldn't* opt in. A concurrent state write (hook / second session) between its read and write is silently lost (`atomicWrite` prevents a torn file, not a lost update). Both reviewers flagged it. **Fix:** exported `withStateLock` from `state.js`; wrapped the whole evict read-modify-write in it. Only bites the parallel-session case the project norm forbids, but STATE.md is *the* lock-protected resource. ~6 LOC.

### I-4 — FUTURE-IDEAS ledger dedup-key collision → silent cross-run **data loss** — **FIXED**
`tools/lib/drain.js` `evictionKey`. Key was `sha1(heading + '|' + dateISO)` — not unique. Two distinct entries sharing a heading+date (e.g. a date embedded in the heading), with one ledgered in a *prior* run, made the second get spliced from the inbox but never appended (additions empty) → **lost from the move-never-delete archive** (the Epic's headline safety NFR). **Reproduced:** BODY-B vanished from both inbox and ledger. **Fix:** key on the block **body** (`sha1(content.slice(range))`) — preserves crash-idempotency (a re-run's un-removed block is byte-identical → same hash → dedupes) while distinguishing distinct bodies; plus a within-run `staged` set (kills the S-1 duplicate too). Post-fix both bodies survive in the ledger. Regression test added. ~8 LOC.

## Security Findings (OWASP/ASVS lens, narrowed to path/injection/data-integrity — no network surface)
- ✅ **Path traversal into the archive is structurally impossible.** `deriveEpicArchiveDir`'s anchored regex emits only `.planning/archive/[M0-9.]/E[0-9]`; crafted `epicId` (`../../etc`, absolute, `M5..5.E1`) all throw. **Triple-guarded** (`EPIC_ID_STRICT_RE` in extractEpicSection, `deriveRetroPath` re-validate, `deriveEpicArchiveDir`'s own regex) — and the belt-and-suspenders `startsWith` check is now added too.
- ✅ **No regex injection / ReDoS.** `extractEpicSection` validates then escapes `epicId`; the heading pattern is O(n²) worst case over a size-capped input.
- ✅ **STATE.md eviction loss-proof by construction** — archive-first + byte-identical assert + crash re-run no-ops (pointer is a `- ` item, not a heading). `before+section+after` tiles the body gap-free.
- ✅ **Safety gates fail closed** (lossy-card refusal, not-closed refusal, `readState` ahead/missing-schema); advisory paths fail open correctly (size banner, `detectMode`).
- ✅ **git shell-outs (pre-existing, unchanged) hardened** — `execFileSync` args array, `last_updated_commit` validated by `COMMIT_TOKEN_RE` before any `A..B` range. 0 new dependencies; no manifest change.
