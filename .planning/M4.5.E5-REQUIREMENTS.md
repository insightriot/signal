## Functional Requirements

- **FR1 — Launch-post draft (`docs/launch-post.md`).** ~600–800 words. Leads with the 7-plugin landscape analysis (D-E5-9) → Signal as synthesis + calibration router. Heritage-respectful; no over-claiming; **no fabricated validation** (no invented user counts, testimonials, or benchmarks). Links to `analysis/REPO-ANALYSIS.md`, `docs/vs.md`, and `examples/url-shortener/`. Drafted by Claude; published by Brett (publish-when-ready asset).

- **FR2 — Demo recording script (`docs/demo-script.md`).** A turnkey storyboard for a ~30–60s screen recording: exact command sequence (the E3-deferred `/sig:init` → `/sig:status` walk, or a tight calibrate→ship snippet), narration beats, and what each frame should show. Reproducible by anyone with the plugin installed. Claude scripts it; Brett records.

- **FR3 — Tester brief + friction-log template (`docs/tester-brief.md`).** A short invitation for a peer to run `/sig:init → /sig:ship` on their own real work, plus a structured friction-log template (where did you pause / what confused you / what failed / what worked / what would you change). Requests **nothing sensitive**. Designed for async use; results feed the v0.1.(N+1) backlog.

- **FR4 — Who-to-ask guidance.** Lightweight criteria for a good first peer tester (not the author; a real project; Mac for now, since Linux/WSL install is still unverified per shelved E1). May be folded into FR3's brief rather than a standalone file — decide in PLAN.

- **FR5 — Version-decision rubric.** A minimal, reusable rubric for the v0.1.x → v0.2.0 call (what counts as a minor vs. patch for Signal). **PLAN must decide its location:** a standalone `docs/versioning.md` (which would partially fulfill the shelved E1 versioning-policy work — flag the overlap, do **not** balloon into all of E1 Slices 3–5) vs. a short section in the Epic artifacts. Either way it produces the written basis for D-E5-4's call.

- **FR6 — Release-notes draft.** GitHub release notes drafted from the CHANGELOG for the version E5 cuts (cumulative E4 + E5 + M4.5 surface), linking the launch post. This is the spec's minimum-bar launch artifact ("a GitHub release announcement linking to the post").

- **FR7 — Human-handoff checklist.** The outward actions Brett runs async (D-E5-6): recruit N peers, record the demo, publish the post, push the release/tag, capture friction logs. Committed so the human-gated tail of E5 is explicit and trackable, not lost on Epic close.

- **FR8 — CHANGELOG entry + validator/test wiring.** An E5 entry under the current `[Unreleased]` block (version stamped at close, D-E5-5). If any new doc should be validator-registered (as `docs/vs.md` was added to `REQUIRED_FILES` in E4), decide in PLAN and update `validate-plugin.js` + its test. No new runtime dependencies.
