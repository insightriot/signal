## Anti-Rationalization Table

Per FULL `gate_strictness: strict`. Surface every temptation explicitly; require explicit justification to skip.

| Temptation | Check |
|---|---|
| Skip `homeDir` injection — fixtures are obviously synthetic | NO. D-E8-10. `os.homedir()` in production code paths means tests can accidentally execute destructive scripts against the dev's real `~/.claude/`. Encode isolation in the code. |
| Skip Signal-scoping detectors — the cache dir is plugin-scoped | NO. D-E8-11. `settings.json` and `installed_plugins.json` are NOT plugin-scoped; they contain entries for every installed plugin. Detector that doesn't filter would propose `rm -rf` against other plugins. |
| Treat the GitHub API as authoritative | NO. The API can 404, timeout, rate-limit, return malformed JSON. Treat as advisory; on any failure print briefing without staleness banner. AC #7 tests mock fetch — NEVER hit live network. |
| Use `semver` as a new runtime dep | NO. Hand-roll 3-part numeric compare (~25 LOC). Signal's only runtime dep today is `yaml`; doubling it for a 4-input function is unjustified. |
| Use `sed`/`grep` for JSON edits in the generated script | NO. JSON has nested structure; `sed` corrupts. Use `node -e` with parse-mutate-atomic-write. |
| Skip the dogfood (AC #13) — Mac Studio is "real" enough as dev | NO. AC #13 is the verification that `/sig:doctor` works against a real install, not synthetic fixtures. Back up first, then run `--fix` on Mac Studio's live P2 + P4. |
| Auto-execute `--fix` because it's surgical | NO. D-E8-1 (DISCUSS) locked script-gen-with-review for BOTH flags. Consistency over per-flag UX nuance. |
| File P1/P2 upstream issues as new — even if duplicates exist | NO. D-E8-9. Cross-link existing #56740 + #62497 instead. Duplicates embarrass Signal in front of E5's launch audience. |
