## What changed mid-flight
- **S6a was split A/B at the arming boundary.** The advisor moved the cut from t2|t3 to t3|t4: building the v3 chain *correct* (rename + evict + BACKLOG + `isV3Conformant` stamp-at-tail) had to stay one unit while the version constant stayed 2 (inert on real repos, fixtures-only) — the version bump 2→3 is the arming commit that makes it fire. Verifying the mechanical chain green *before* arming is the right safety posture.
- **The stamp-null rollout gap became a fix, not a footnote.** VERIFY-adjacent analysis surfaced that `needsV3` excluded stamp-null — and *every* existing external project is stamp-null at the combined release (the layout stamp is unreleased). Brett's call: fix it (converge existing projects), not ship resolver-back-compat-only.
- **The B19 release-gating question resolved to "fix it."** REVIEW framed it as a release-gate decision; the panel showed a ~20-LOC parity+guard moots the data-loss severity, so it was fixed in-phase rather than shipped as a documented limit.

## What assumptions broke
- **"Green fixtures prove the migrate is faithful."** The live dogfood — not a fixture — caught **two** bugs the 1400+ green tests couldn't: (a) the D-ID map homed `D-v016-2` by a live *reference* instead of its archived *definition*, so the fail-closed anchor gate correctly **refused the whole eviction**; (b) the tail index-regen **clobbered** Signal's own curated INDEX (old table format the parser can't read). AC6.5's "dogfood on real memory" is the exact control that green tests structurally cannot be — it earned its place.
- **"The dry-run shows what apply does."** B19: `renderDryRun` claimed "enumerates EXACTLY the steps apply performs" but omitted the INDEX regen it runs — a trust defect in the one command whose entire safety model is *human-approves-the-preview*.
- **"A minimal migrate run is safe by construction."** The security audit found the planning-root realpath check fired only through a relocate/archive vector; a minimal `needsV3` run reached the first write with no symlink check — the exact stranger-adoption case the stamp-null fix made universal.

## What surprised us
- **The fail-closed gate did its job on live data on the first apply.** The D-v016-2 miss wasn't a test — it was the real dogfood apply refusing to write and rolling back byte-identical. The "detect-only if <100% resolve" design (R4) turned a latent silent-degradation into a clean, diagnosable stop.
- **The definition-vs-reference bug was the S2 range-membership caveat, made concrete.** It was flagged as a *theoretical* caveat when S2 shipped; the dogfood turned it into a one-line reproduction (`D-v016-2` defined in an evicted section, referenced in a live one).
- **The panel's rigor caught what a rubber-stamp wouldn't.** The test reviewer checked out an old worktree to *empirically* confirm the disclosed stash-verify RED was genuine, and confirmed no arming-ripple assertion was weakened — the exact "flip the test to pass" trap, cleared with evidence.

## What we'd do differently
- **Dogfood the risky migrate on a *copy* of real memory before the real apply, earlier** — the D-ID and INDEX-clobber bugs were both discoverable from a throwaway-copy dry-run+diff; running that at S5/S6a (not S6b) would have surfaced them pre-arming.
- **Make the dry-run's step enumeration a test invariant, not prose.** B19's parity gap existed because "dry-run == apply steps" was a comment, not an asserted property. A test that diffs the dry-run's step list against apply's recorded `moves` would have caught it.
- **Treat "runs on every project" as a threat-model trigger.** The stamp-null fix widened the migrate's blast radius to all external repos; the symlink gap (IMPORTANT-1) and B19's general-project exposure both trace to that. A "who does this now run on?" check when a gate widens would front-load those.

## What to feed back into Signal
- **AC6.5-style real-memory dogfood is non-negotiable for destructive doc automation.** It caught two bugs an exhaustive green suite + an independent 28/28 AC verifier both missed. The human-approved dry-run diff is the control; the dogfood is the proof.
- **The 3-specialist adversarial panel keeps paying for itself** — same as M5.E2's SHIP-blocker, this Epic's panel found a trust defect (B19 parity), a symlink escape, and a Nyquist coverage gap that "tests pass" hid. Worth being the default REVIEW shape for anything that mutates user files.
- **Fail-closed + "route the safe default" is the right posture for cross-project generality that isn't fully built yet** — detect-only append-log evict for external projects (until the milestone-map generalizes) shipped the mechanism safely without pretending it's universal.

## Anti-rationalization moment
Two pulls, both refused. First, at the dogfood: the migrate clobbered the curated INDEX and the temptation was to accept the auto-generated `_(note pending)_` version as "good enough, it's accurate to structure" and move on — but that silently drops load-bearing gotchas a stranger has no one to re-port, so the notes were hand-ported and the gap was logged (B19) for a real fix. Second, at REVIEW: with B19 framed as a release-gating question, the easy path was "document it as a known limit and ship" — the honest read was that a ~20-LOC parity+guard *removes* the risk, so it was fixed in-phase, not deferred. The through-line: "the dogfood proved it works" is not "the dogfood was clean" — it surfaced two real bugs, and the discipline was to fix them before the SHIP conversation, not to let a successful headline paper over them.
