### 5.1 PLAN-deviation log (3 surfaced, all resolved in-flight)

1. **S1.t11 threshold formula vs. AC.** PLAN's `150B × section_count` coefficient is incompatible with the PLAN AC "minimally-filled (one sentence per section) passes." Resolved with 60B coefficient honoring the AC; deviation documented inline + in commit `5ed3898`. **For REVIEW:** consider promoting "spec-internal consistency" to a standard PLAN-validation dimension so this class doesn't recur.

2. **S1.t7 bash → Node hook wrappers.** PLAN spec called for `hooks/check-state-write.sh` (bash). Collapsed to direct Node invocation (`hooks/check-state-write.js`) for cross-platform reasons (Windows lacks bash). Documented in commit `d38e210`. Validator green; hooks.json correctly references the .js files.

3. **S1.t9 → S1.t6 sequencing.** RESEARCH § 1.1 surfaced that `commands/ship.md` had a state-write gap relative to verify.md/review.md. The fix (programmatic `transitionPhase` + `markFresh`) bundled into S1.t6 per PLAN § 1.5 "PASS-WITH-NOTE" scope-by-discovery callout. No downstream impact.

### 5.2 Limitations surfaced but unresolved (not in scope for E9)

1. **MILESTONE-4.5.md does not yet have an E9 row.** The status snapshot table at the top of `MILESTONE-4.5.md` (lines ~27-44) currently lists E1-E8 but not E9. This means `isEpicCloseShip` for `current_epic: M4.5.E9` will return `false` until the row is added. **This is appropriate Epic-close work, not retro work.** The row should be added as part of M4.5.E9's eventual SHIP commit, with status "shipped 2026-05-DD" matching the convention.

2. **SessionStart-resume hook manual smoke not yet run.** The unit tests (`tests/hook-state-write.test.js`) cover the JS surface of `detectDirtyExecute`. The hook-firing handshake (Claude Code → SessionStart event → hooks.json matcher: resume → `node warn-dirty-execute.js` → stdout JSON → context injection) is not yet end-to-end verified in a real session. The PreToolUse hook was manually smoke-tested 2026-05-26 (confirmed exit 2 + stderr block); SessionStart-resume was not. **Flagged for the user to validate when convenient** by triggering a session resume against a synthetic dirty-EXECUTE STATE.md.

3. **Cross-runtime adapter parity (Cursor / Codex).** Documented as known limitation in PLAN. Cursor users get command-internal-only (no hook surface for PreToolUse on file writes); Codex gets the full layered stack. Out of E9 scope.

4. **Stub-completion is not an Epic exit criterion (A7).** Per D-E9-7, the 5 backfilled stubs may sit at `[FILL IN]` indefinitely. This is by design — the Epic's value is forward-only enforcement, not retroactive backfill. Surfaced for awareness.

5. **`current_wave: 4` in STATE.md.** STATE.md frontmatter still shows `current_wave: 4` from Wave 4 of S1; was not bumped to `5` for the S1.t12 wave or any of S2. **Minor — for SHIP-time cleanup.**
