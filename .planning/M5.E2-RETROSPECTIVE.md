# M5.E2 — Auto-sensing migrate command — Retrospective

**Outcome:** shipped `/sig:migrate-memory` (FR6/FR7) — the risky, go-big piece of the doc-runtime flagship. Auto-senses an old-layout project, plans the smallest safe move, and (on explicit confirm) reorganizes its `.planning/` **relocate-never-delete, dry-run-by-default, git-reversible, idempotent**. Handles all three bloat vectors (v1 frontmatter-prose de-prose, v2 whole-body relocate → `STATE-HISTORY.md`, v3 closed-Epic narrative eviction), the archive-tree scaffold relocation, and the FR7 doc-layout stamp (`docs_layout_version`) + fail-open SessionStart / resume / status banner. Proven on the live pain: `nextpass/.planning/STATE.md` **546 KB → 1.3 KB, 0 words dropped** (re-confirmed on post-REVIEW HEAD), and dogfooded on Signal's own `.planning/` (31 archive relocations + stamp v2). ~2,724 production lines. FULL/strict, full DISCUSS→SHIP. ≈1071 → **1300 tests** (+229). Decisions D-M5E2-1…6 (DECISIONS 2026-07-16 / 2026-07-18).

**Decisions with IDs (the card):** the E2/E3 pivot pulled the migrate command forward from E3 (D-M5E2-1) because live bloat was blocking ~5 projects; scope = the *complete* command, not a narrow first cut (D-M5E2-2); the deliverable is releasable (D-M5E2-3), but D-M5E2-6 later folded E3 in so the doc-runtime ships as ONE `0.1.x` = E1+E2+E3; de-prose **relocates, never deletes**, gated by the model §5 word-accounting + human dry-run eyeball (D-M5E2-4); conservative auto-sense across unseen projects — smallest safe move, re-run is a no-op (D-M5E2-5).

**Still open (carried forward):**
- **Ticketed fast-follows (none SHIP-blocking):** **B14** (codebase-wide lexical symlink confinement in `evict.js`/`add.js`/`resume.js` — same fix as I-b, out of E2's attack surface per §9), **B15** (>1 MB single-file scan-ceiling can silently defeat the dangling gate — extreme; real corpora <1 MB), **B16** (a rolled-back `--apply` leaves a lingering `pre-migrate-memory-<stamp>` tag).
- **New at SHIP:** **B17** — 4 git-heavy migrate tests flake on vitest's 5 s default under full-suite parallel load (pass clean in isolation / on re-run). Test-infra only; cataloged, not fixed in SHIP.
- **Release gating (D-M5E2-6):** the marketplace release is **not** cut here. M5.E2 lands on `main` intentionally unreleased; the combined **E1+E2+E3** release cuts after M5.E3 completes its own DISCUSS→SHIP. ## What changed mid-flight
- **The whole Epic was a pivot.** E2 was going to be the all-docs hygiene runtime; live doc-bloat wedging `nextpass` (529 KB STATE, B8) reordered the milestone at DISCUSS — the migrate command that un-sticks *existing* projects was pulled forward and the hygiene runtime moved to E3 (D-M5E2-1).
- **The release grew after REVIEW.** D-M5E2-3 had E1+E2 releasable on its own; D-M5E2-6 (Brett's call, post-REVIEW) folded E3 in so the doc-runtime ships as one battle-tested unit rather than "eviction shipped but the append-logs like `DECISIONS.md` still bloat." That turned this SHIP from a release into an unreleased land-on-`main`.
- **B12 was a dogfood find, not a planned task.** The real nextpass migrate (not a fixture) filed a `▶ Active:` marker into history; caught by the eyeball the model §5 exists to force, fixed mid-VERIFY (`e42afed`).

## What assumptions broke
- **"Green tests mean the migrate is safe to apply."** The Critical (C1): rollback was **not** wired around the mechanical move/rewrite phase — a routine I/O throw mid-apply (ENOSPC/EACCES/ENOTDIR) left STATE de-prosed+stamped, scaffolds half-moved, referrers half-rewritten, and in `fs-backup`/`--force` modes the durable snapshot persisted only *after* the throw → **zero recovery**. The full suite was green the whole time because no test injected a throw inside that window. Two reviewers reproduced it independently (`.planning/archive` as a regular file → ENOTDIR). The SHIP-blocker lived exactly where the tests couldn't see.
- **"Conformant frontmatter == what the real parser accepts."** The migrate's regex-"conformant" output was never asserted against `state.js`'s real `yaml` consumer that resume/status/doctor use (I-c). The round-trip test added in REVIEW **passed for all 5 shapes** — safe by construction, but the gap was "unasserted," now closed.
- **"`completed_phases` entries are always closed phases."** B12 — an active-work marker parked there got swept to history; the fix derives a meaningful label + flags non-standard entries in the dry-run rather than silently relocating them.

## What surprised us
- **Two independent reviewers reproduced the same Critical.** The code-reviewer and security-auditor, working blind to each other, both found and empirically reproduced the rollback gap — high-confidence convergence, which is rare and made the SHIP-blocker call unarguable.
- **The fix volume blew past the 50-LOC PASS-WITH-FIXES rule-of-thumb** (5 batches, ~19 new tests in REVIEW alone) — yet not one finding required re-planning or a loop back to EXECUTE. Volume ≠ re-plan.
- **The test suite is timeout-flaky under its own weight.** 4 git-heavy migrate tests exceed vitest's 5 s default when 81 files run in parallel; they pass clean in isolation and on a second full run. Surfaced only at SHIP → B17.

## What we'd do differently
- **Wrap every state-mutating I/O in the rollback boundary from the first EXECUTE task**, not just the git operations. The Critical was a "which lines are inside the try/catch" oversight; a one-line "what happens if line N throws?" checklist over the apply orchestrator at PLAN would have caught it before REVIEW.
- **Assert against the real parser the moment a plan says "conformant output."** Don't let "conformant" mean "the regex looks right" — round-trip through the actual downstream consumer.
- **Give git/fs-heavy tests an explicit timeout on first write.** The B17 flake was predictable the moment those tests started shelling out to real `git`.

## What to feed back into Signal
- **The 3-specialist adversarial panel + independent reproduction earns its cost on any command that writes/moves/deletes user files.** It caught a SHIP-blocker that VERIFY (which checks code-correctness, not adversarial inputs / injected failures) structurally could not. Worth making the default REVIEW shape for destructive automation.
- **"Relocate-never-delete + human-approved dry-run diff" is the right contract for destructive automation** — the nextpass dogfood (0 words dropped, eyeballed) is the proof that green tests alone can't give.
- **RED-first-per-finding kept fix-in-phase honest.** Each of the 5 REVIEW batches proved the bug before fixing it; without that, "we fixed it" would have been an assertion, not a demonstration.

## Anti-rationalization moment
With the fix volume past the PASS-WITH-FIXES threshold, the temptation was two-headed: either downgrade the verdict to "loop back to EXECUTE" (ceremony that would have stalled a fully-fixed Epic), or wave off the rollback Critical as "unlikely in a real repo" and ship. Both refused. The Critical was a genuine unrecoverable-partial-write in a stranger's `.planning/`, reproduced twice; it was fixed in-phase, RED-proven, **before** any SHIP conversation. The honest framing — "high fix volume, but no finding needed a re-plan" — is the workflow's in-phase-fix pattern, not a bent rule. And this SHIP itself resisted the "it's done through REVIEW, just cut the release" pull: D-M5E2-6 says the doc-runtime isn't *complete* until E3 extends hygiene to the append-logs the migrate protects but doesn't clean — so this lands unreleased and waits.
