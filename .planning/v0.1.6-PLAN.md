# v0.1.6 — Doc-Integrity Guardrail — Plan

**Goal:** prevent new documentation-integrity pathology at write time and converge two capture/planning papercuts — without new deps, keeping 854 tests green.

Design detail + risk analysis live in `v0.1.6-RESEARCH.md`; this plan is the executable slice list and references it rather than restating it. Spec: `v0.1.6-REQUIREMENTS.md`. Decisions: `DECISIONS.md` § 2026-07-13.

## Architecture decisions (locked)

1. **FR1 discriminator is field-specific, raw-text, blacklist.** `completed_phases`: block a multi-line raw item OR one > `COMPLETED_PHASES_MAX` (150). `blockers`: never flag multi-line (a real blocker object is 3 lines); block only its `text:` value if multi-line-scalar OR > `BLOCKER_TEXT_MAX` (500). When in doubt, **allow** (false-positive wedges a stranger = P1; false-negative just defers to FR2). Raw-text, not `parseFrontmatter` (which throws on the target input → fails open).
2. **FR1 lives in a sibling pure fn**, not inside `checkProposedStateWrite` (which gates on SHIP). The hook composes two predicates, OR-combines `block`, `reason` names which fired. `checkProposedStateWrite` stays byte-identical (AC1.4).
3. **FR1 is self-unblocking** — stateless hook; a cleanup edit to clean frontmatter always passes (AC1.7). This is why R1.2 = full-content block (a), not delta-only.
4. **FR2 measures whole-file `statSync`** (read-only, catches frontmatter bloat too), threshold `STATE_SIZE_WARN_BYTES` (150 KB); mirror the schema-drift `detect/read/format` trio; render below the trust banners. Tests use **controlled-size fixtures, never the live STATE.md**.
5. **FR3 regex is `^`-anchored + fence-aware**, verb set `Promoted|Deferred|Merged|Shipped|Deleted` (not `Update`); frozen `listDrainCandidates` contract; additive (no snapshot break).
6. **FR4 clause boundary** = `— . : ,` **only when followed by whitespace/EOL** (URL guard), em-dash only (not hyphen), min-length floor ~20 chars/3 words, 60-char cap retained as fallback.
7. **FR5 deletes by explicit line-range** (heading → closing `---`), never `applyDisposition` (would swallow the footer); assert `lintFutureIdeasFooter → {ok:true}` after.

All new numeric limits are **named constants**, not literals.

## Slices

### S1 — FR1: block prose in STATE frontmatter  ·  M  ·  (P1, gets REVIEW pass)  ·  deps: none
New sibling pure fn (e.g. `checkStateFrontmatterShape({ proposedContent })` in `tools/lib/retrospective.js`, exported); hook calls it + the existing check, blocks if either. Raw-text field-specific discriminator per AD1; fail-open on any parse/throw.

**Acceptance criteria**
- AC1.1 — `completed_phases` item that is multi-line OR > 150 chars → block; reason names `completed_phases`.
- AC1.2 — `blockers[].text` that is a multi-line block scalar OR > 500 chars → block; reason names `blockers`.
- AC1.3 — well-formed write (compact `completed_phases`, `blockers: []`) → allow. Locked passers: 58-char annotated phase entry; Signal's own current frontmatter.
- AC1.4 — existing retro-absence SHIP block still fires unchanged (no regression — the 6 current `checkProposedStateWrite` cases stay green).
- AC1.5 — fail-open: malformed hook input / non-STATE path / non-Edit-Write / frontmatter that makes `parseFrontmatter` throw → exit 0.
- **AC1.6 (new)** — a legit **multi-line `blockers` object** (`id`/`text`/`raisedAt` on separate lines, `text` under budget, e.g. the orphan fixture, incl. a `text` with a semicolon) → **allow** (never flagged for being multi-line).
- **AC1.7 (new)** — a write producing **clean** frontmatter is **allowed regardless of the file's prior polluted state** (self-unblock).

**Tests** (`tests/hook-state-write.test.js`, pure-fn + DI): AC1.1/1.2 use the cmmc frontmatter as the malformed fixture; AC1.3/1.6 lock the legit-passers; AC1.4 = existing suite; AC1.5 feeds throw-y input. **Files:** `tools/lib/retrospective.js`, `hooks/check-state-write.js`, `tests/hook-state-write.test.js`.
