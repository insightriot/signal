## Functional Requirements

### FR6 — Auto-sensing migrate / re-org command — **CRITICAL PATH, the one high-risk surface**

A standalone command (name provisional **`/sig:migrate-memory`**; **not** `/sig:doctor`, which stays install-only) that reorganizes the **invoking** project's docs to the FR1 model. `tools/archive-migrate.mjs` is the existing prototype to extend.

- **FR6.1 — Dry-run by default.** Prints the full plan — every move and every rewrite, with before/after — and changes nothing. Apply requires an explicit confirm.
- **FR6.2 — Git-backed + rollback.** Refuses to run on a dirty working tree without an explicit acknowledgement; every apply is a single revertable unit (or records pre-state for rollback).
- **FR6.3 — Move-never-delete + dangling-link verification before AND after.** No closed content is ever deleted; the original survives in archive; no link is broken by a move.
- **FR6.4 — Idempotent.** Re-running on an already-migrated project (FR7 stamp present + structure conformant) is a **no-op**.
- **FR6.5 — Conservative auto-sense (advisor, 2026-07-16).** Detects the old layout from structural signals + the FR7 stamp. It **never assumes a uniform old layout** — it handles varied sizes, **partial** migrations, and linear vs Epic mode. When unsure, it plans the *smallest safe move* and flags the ambiguity in the dry-run; it never guesses destructively on a project it can't inspect ahead of time.
- **FR6.6 — Three bloat vectors (model §3), each RELOCATE-not-delete:**
  - **Vector 1 — frontmatter-list prose** (the acute 529 KB `nextpass` case). De-prose = **relocate** the narrative out of the YAML list into the STATE body / an archive card, leaving a short scalar. **Never delete it.** ⚠ **Headline risk:** the B8 hand-recipe *deleted* the prose (frontmatter 107→27, "body byte-identical" = the ~80 lines were dropped). Fine for Brett eyeballing one file; **catastrophic as an unattended default** — it's the silent-memory-degradation failure model §5 exists to prevent, and it breaks move-never-delete. The automated command **must relocate with the §5 word-accounting + faithfulness gate.**
  - **Vector 2 — inlined legacy body** (E1 did this with a one-off hand script; E2 makes it a **reusable general helper**): relocate the already-migrated inlined body → `STATE-HISTORY.md` + one-line pointer.
  - **Vector 3 — ongoing body accretion** (closed-Epic narrative): E1's evict-on-close handles *new* closes; E2's migrate applies the same eviction **retroactively** to a project's backlog of already-closed Epics (distill card + pointer + archive).
- **FR6.7 — Faithfulness gate on every relocation (model §5 ordered gate).** distill (lift-verbatim, keep IDs, lift open items up) → **verify-against-source** (deterministic coverage backstop: every `M*.E*`/`D-*`/`FR*`/`AC*` ID, ISO date, and `open`/`deferred`/`shelved`/`carry-over` status token survives, or lands on an explicit dropped list) → **then** evict. A lossy relocation **fails** the gate.

### FR7 — Doc-layout version stamp + upgrade banner

- **FR7.1** — a `docs_layout_version` integer in STATE.md frontmatter, its **own axis** (distinct from `schema_version` and the plugin's release SemVer). Absent/`1` = pre-reorg; the migrate command sets it to the current layout version on completion.
- **FR7.2** — a **SessionStart** Node hook (reusing `state.js`'s frontmatter reader, matching `check-state-write.js`) + `/sig:resume` + `/sig:status` emit a one-line banner when a project is pre-reorg on a post-reorg plugin ("docs predate the vX.Y.Z layout — run the migrate command, dry-run first"). Post-reorg → **silent**. Reads the stamp first (fast), cheap structural sniff fallback. **Fail-open** — malformed/absent never crashes a session. Fires in every installed repo.

## Non-Functional Requirements

- **Safety-first (headline NFR).** No data loss; **dry-run is the default**; move-never-delete; every apply git-reversible; dangling-link verification pre/post. This command's entire job is to touch real project docs — these are non-negotiable.
- **Faithfulness — the risk safety-first does NOT cover.** Relocation replaces live narrative with a card + pointer; mechanical safety preserves the original, so the residual risk is *legibility degradation* from an unfaithful card. **Tests cannot prove faithfulness** (model §5 blind spot). So the faithfulness proof is a **human-approved per-project dry-run diff** (the confirm step shows the actual before/after content) **+ the Signal-own dogfood eyeball — not green tests.** PLAN must build the confirm UX around this, not around a passing suite.
- **Operates on the invoking project.** Reads/writes the CWD project's `.planning/` + docs, never Signal's repo (except when Signal is the invoking project). No hard-coded Signal paths.
- **Backward-compat / no forced migration.** A project that never runs it keeps working; the banner nudges, never blocks. Both linear and Epic mode handled.
- **Signal-native.** No runtime dependency on the external Curator CLI (DECISIONS 2026-07-13).
- **Cross-platform, fail-open hooks** (mirrors the E10/E11 hook posture; Windows-safe; never crashes a stranger's session).
- **FULL-tier rigor.** TDD, strict Nyquist (every AC maps to a test), full REVIEW panel — this touches the state core every installed repo depends on **and** rewrites user docs.
- **Production ops NFRs — N/A.** Health/liveness probe, graceful-shutdown, structured request logging, security headers, rate limiting: N/A — local doc-tooling / CLI + hooks, no network-service surface. Recorded explicitly per the FULL-tier NFR prompt.
