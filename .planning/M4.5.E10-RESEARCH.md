## ⚠ Three PLAN-gate scope decisions (surfaced, not silently resolved)

Research turned up three places where the DISCUSS requirements assumed something the source contradicts. Each is a product/scope call for the user, gated individually under FULL/strict before the plan finalizes.

### SD1 — FR1 (`current_epic`) is inert for strangers; it's a Signal-on-Signal dogfood fix

**Verified in source:** `current_epic` is initialized to `null` (`state.js:167,213`) and **never assigned a non-null value by any command or helper** — every other reference reads it (`retrospective.js`, `backfill-retros.js:405`, `warn-dirty-execute.js:39`). Signal's own `STATE.md` shows `M4.5.E10` only because it was **hand-edited** in the DISCUSS commit `c031a3a`. Separately, `commands/plan.md`/`discuss.md` write **phase-prefixed** artifacts (`.planning/{phase}-PLAN.md`, `.planning/REQUIREMENTS.md`), **not** Epic-prefixed. So for a command-driven stranger project: artifacts aren't Epic-prefixed **and** `current_epic` is null → pattern-0 never fires; legacy patterns 1–3 already resolve their files. **FR1's real beneficiary is Signal-on-Signal** (hand-managed Epic-prefixed artifacts + hand-set `current_epic`), where `/sig:resume` genuinely can't find `M4.5.E10-PLAN.md` today. That's a real, daily maintainer papercut — but it is *not* the stranger-facing fix the REQUIREMENTS framed. Making it stranger-facing would need Epic-as-a-first-class-command-concept (a `current_epic` writer **and** Epic-prefixed artifact writing) — M5 / multi-feature-lifecycle territory, explicitly barred by M4.5's "no new architectural surface." → **Recommend: keep FR1 as the resolver-only fix, honestly scoped to Epic-prefixed/hand-managed projects.**

### SD2 — FR5 doesn't fit `/sig:doctor`: it's `~/.claude`-only, macOS-gated, and `SCHEMA_VERSION` isn't exported

**Verified in source:** every `doctor.js` function takes `homeDir` (the `~/.claude` install root); none reads a project `.planning/STATE.md`, and `doctor.js` never imports `readState`. `commands/doctor.md` Step 1 runs `checkDoctorEnvironment`, which **exits with a polite stub on non-macOS before any detection** — so a Linux/WSL tester (the exact upgrade-seam audience) would get **no schema warning**. Schema drift is a project-file concern with zero dependence on macOS or `~/.claude`; inheriting the install-diagnostician's platform gate is a category error. It also contradicts E8's charter (`MILESTONE-4.5.md:252`: "`/sig:doctor` is strictly about install/upgrade state in `~/.claude/`"). Plus `SCHEMA_VERSION` is module-private (`state.js:97`; export list at `:366` is only `PHASES, PLANNING_DIR`) — assumption A3 ("doctor can read it directly") is currently **false**; a one-line export is a prerequisite. → **The user's D-E10-3 gate chose the *content* (schema-drift focus); the *host* is now the open question.** Options in the gate below.

### SD3 — the privacy claim is already inaccurate, and E10's git-fetch widens the gap

**Verified in source:** `README.md:183` + `docs/launch-post.md:27` state "Signal makes **no network calls beyond what Claude Code itself makes to Anthropic's API** … no telemetry, no usage pings." That is **already false** — E8's version check hits the GitHub tags API (`doctor.js:539`) for any user running `/sig:status`; the `audit-network-calls.js` regex (`/\bfetch\s*\(/`, `tools/audit-network-calls.js:51`) stays green only because E8 routed through a `fetchFn` indirection that dodges it. E10's `git fetch origin` (via `execFn('git', ['fetch', …])`) is a **second** network category that the regex also won't catch. Given the "docs always 100% accurate" norm **and** E10's audience explicitly asks about privacy, shipping git-fetch without reconciling the claim is a documented defect. → **Recommend: add a doc-reconciliation task to E10** (state the two real network calls accurately — GitHub version check + read-only `git fetch` against *your own* remote, no data sent), and optionally extend `audit-network-calls.js` to catch git-fetch shell-outs so the contract is enforced, not regex-evaded.

## Per-item implementation findings

### FR1 — Epic-prefix artifact resolver
- **Add** `resolveArtifactPath(planningDir, artifact, {currentEpic, phase})` → absolute path or `null`, in `tools/lib/resume.js` (today it exports only `renderResumeBriefing`/`handleOrphansAtResume`; resolution is prose in `resume.md:55–61`). Precedence 0→1→2→3, first `existsSync` hit wins.
- **Pattern 0** = `${currentEpic}-${artifact}.md`, built **only when `currentEpic` is a non-empty, sanitized string** (AC1.2).
- **Sharpest risk — path traversal:** `current_epic` is stranger-editable YAML; a value like `../../etc/x` escapes `.planning/`. Mirror `add.js`'s path-confinement guard (`resolve()` + `startsWith(planningRoot + sep)`, ~`add.js:620`); validate `currentEpic` against `/^[A-Za-z0-9._-]+$/`, and fall through to legacy on reject. **Add a traversal test.**
- Empty-file match → existence-only resolve (empty section is cosmetic, not a failure). Multiple legacy matches → deterministic ascending-N. Mirror `deriveRetroPath`'s Epic-ID regex (`retrospective.js:127`).
- **Tests:** pure over an in-memory `files` list (mirror `resume-briefing.test.js:37–63`) + one on-disk fixture `tests/fixtures/resume/epic-prefixed/.planning/M4.5.E99-PLAN.md`.
- **One consumer only:** `/sig:status` and `/sig:checkpoint` deliberately do **not** read phase artifacts (confirmed in `status.md`/`checkpoint.md`) — extract for *testability*, do **not** thread the resolver into siblings (would break their lean "snapshot, don't read artifacts" identity).

### FR2 — Origin-drift (`isStaleVsOrigin`)
- **Add** `isStaleVsOrigin(baseDir, {execFn})` in `tools/lib/state.js`, sibling to `isStateStale` (610–653); reuse its `execFn` default + `stdio:['ignore','pipe','ignore']` + try/catch→benign-default+stderr pattern. Returns `{stale, aheadCount, commits[], touchedPlanning}`.
- **Validated git sequence (empirically, git 2.39.3):**
  1. `stored = state.last_updated_commit`; null → `{stale:false}`.
  2. Default branch: `git rev-parse --abbrev-ref origin/HEAD`. **Gotcha:** on unset `origin/HEAD` it prints literal `origin/HEAD` to stdout **and exits 128** — safe under `execFileSync` (nonzero throws), but also guard `output === 'origin/HEAD'` → fall back to `main` (AC2.5). `tracking = 'origin/'+branch`; `branchName = branch` for the refspec.
  3. Bounded, non-interactive fetch: `execFn('git', ['-c','credential.helper=','-c','core.askPass=','fetch','--no-tags','--quiet','--no-recurse-submodules','origin',branchName], { cwd, timeout: ~2000, killSignal:'SIGKILL', stdio:['ignore','ignore','ignore'], env:{...process.env, GIT_TERMINAL_PROMPT:'0', GIT_ASKPASS:'', SSH_ASKPASS:'', GIT_SSH_COMMAND:'ssh -oBatchMode=yes -oConnectTimeout=2'} })`. Any throw (offline/no-remote/timeout/auth) → `{stale:false}`. **This hardened env is the load-bearing NFR1/NFR2 detail** — the timeout reliably kills a hung child (verified: ETIMEDOUT+SIGTERM at ~timeout).
  4. `git rev-list --count ${stored}..${tracking}` → `aheadCount`; throw → `{stale:false}`; `0` → `{stale:false}`. (Missing `stored`/diverged history → `fatal: bad revision`, exit 128 → catch → fail-open.)
  5. `git log --pretty=format:%H %s ${stored}..${tracking}` → `commits[]`; throw → `[]` (keep count).
  6. `git rev-list --count ${stored}..${tracking} -- .planning/` → `touchedPlanning = n>0` (AC2.3); throw → `false`.
- **Fail-open matrix** (all → `{stale:false}`, never throw): non-git dir, no remote, offline, fetch timeout, `origin/HEAD` unset (→`main`), default-branch-not-`main`-and-`origin/HEAD`-unset (**known fail-open gap — note it**), `last_updated_commit:null`, diverged/force-pushed history. Detached HEAD is irrelevant (we compare `origin/<default>` to STATE's commit, not local HEAD).
- **Optimization (optional):** `git ls-remote origin <default>` sha-compare first (truly read-only, no object download), full `fetch` only when shas differ.
- **Banner:** add an `originDriftResult` param to `renderResumeBriefing` (new block beside the local-staleness banner at `resume.js:68–76`), highlight `touchedPlanning`. **Keep the local + origin banners separate** — different failure modes (D-E10-8), both survive.
- **Wiring:** `resume.md` Step 3b (beside `isStateStale`); `status.md` §2.0 (beside `readStalenessWarning` — the NFR2 timeout is what protects status's ≤30-line contract); `checkpoint.md` §5 readout.
- **Read-only doctrine note (pre-empt REVIEW):** the fetch writes `.git/` (FETCH_HEAD, remote refs), **not** `.planning/` — so `/sig:status`'s "no `.planning/*` mtime changed" read-only gate literally holds. State this in the plan.
- **Tests:** `tests/is-stale-vs-origin.test.js` cloning `is-state-stale.test.js`'s `plantState`+`gitOutput` helpers, but with a **subcommand-dispatching `execFn` stub** (`switch(args[0]/args[1])` over fetch/rev-parse/rev-list/log) — the existing stubs return one value for any call and **no fixture has an `origin` remote**, so this scaffolding is materially new. Timeout leg (AC2.7) tested by making the stub throw an ETIMEDOUT-shaped error (the real `execFileSync timeout` is bypassed under injection — don't over-trust A5).

### FR4 — Capture-pipe guards
- **(a) Drain dangling fence** (`drain.js parseEntries`, fence loop 111–120): today unclosed ``` leaves `inFence=true` → entries below vanish from `listDrainCandidates`. **Surgical recovery:** count fence markers; if **odd**, also collect `## ` headings after the *last* fence-marker line + emit a warning (AC4.1/4.2). Even count → second pass never runs → byte-identical + no spurious warning (AC4.3). **Do NOT** fix fence-type (``` vs `~~~`) matching — pre-existing CommonMark non-conformance, out of scope. **Do NOT** naive "re-parse fence-oblivious + union" (would surface headings inside legit balanced fences).
- **(b) `/sig:add` footer drift** (`add.js insertAboveFooter:424`/`rewriteFooter:405`): the FUTURE-IDEAS path uses **naive first-match** `findIndex(/^\*Last updated:/)`, whereas the milestone path already has the correct trailing-anchored `findTrailingFooterIdx:611`. **Switch FUTURE-IDEAS footer detection to trailing-anchored + fence-aware** (reuse/extend `findTrailingFooterIdx`) — this also fixes a latent bug where a fenced `*Last updated:*` **sample** inside an entry would be matched as THE footer and corrupted. Repair = strip misplaced footers, absorb stranded content above, append one fresh footer at true EOF, land the entry above it, **announce** (AC4.4/NFR4). Well-formed file → no-op except date bump (AC4.5).
- **(c) Lint** (AC4.6): `validate-plugin.js` doesn't scan FUTURE-IDEAS today. Since it validates *plugin structure* and this is *project content* (a Signal-dogfood self-check), house it as a **small dedicated lint/test** (fence-aware: footer = last non-fenced `*Last updated:*`; fail if non-whitespace follows) rather than overloading the structure validator. (Ensure Signal's own `.planning/FUTURE-IDEAS.md` is clean first — it was repaired 2026-06-03.)
- **Design fork resolved (pure vs command layer):** `parseEntries`/`rewriteFooter`/`insertAboveFooter` are contractually **pure, bare-return**. Per api-and-interface-design's "prefer addition over modification," **keep those signatures**; add a sibling `detect+recover` that returns `{content, recovered, kind}` (or a separate diagnostics fn), and put the **announcement in the command layer** — don't hack stderr into a pure module or widen the array return (would ripple to `listDrainCandidates`/`applyDisposition*`).
- **Tests:** inline-string cases in `drain.test.js` (dangling-fence + well-formed no-regression) and `add.test.js` (stranded-content + fenced-sample-false-positive + idempotent well-formed).

### FR5 — Doctor schema-drift diagnostic (host = SD2 gate)
- **Detector must NOT call `readState`** — it throws `StateSchemaError` for **both** the ahead (`>1`) and missing-key cases, indistinguishably (`state.js:263–274`), so AC5.3 "rather than crashing" would be unmet. Instead: `parseFrontmatter` directly (narrow — throws only on malformed YAML), read raw `schema_version`, numeric-compare against the exported `SCHEMA_VERSION`.
- **Prerequisite:** `export { SCHEMA_VERSION }` from `state.js` (currently private).
- **Pure detector** `detectSchemaDrift(rawSchemaVersion, expected)`: null STATE → no finding (AC5.4); legacy no-frontmatter **or** `< expected` → behind → point at the auto-migration path (`references/state-schema.md` + `docs/migration-state-schema-v0.1.x.md`; E6 auto-migrates on first write — E10 detects+points, doesn't migrate); `> expected` → fail-closed report, no crash (AC5.3); `== expected` → no finding (AC5.2); malformed YAML → catch → "unreadable STATE" note. Read-only (AC5.5).
- **Placement is the SD2 gate.** Whichever host: the schema check must be **platform-independent** (run before/beside any macOS `checkDoctorEnvironment` gate) and keyed off the **project cwd**, not `homeDir`. Today only the legacy case is real drift (`schema_version:1` is current) → `>1`/synthetic fixtures for the ahead case (D-E10-3). Tests: in-memory `detectSchemaDrift` cases (mirror `doctor.test.js:26–52`, which uses in-memory objects — **note `tests/fixtures/doctor-states/` does not exist**) + a planted-`.planning/STATE.md` temp-dir case.

### FR6 — SessionStart hook smoke test + `references/hooks-api.md`
- **`warn-dirty-execute.js` ignores stdin** — `baseDir = process.cwd()` (:19), reads `.planning/STATE.md` + `MILESTONE-{N}.md` from disk, output driven by `detectDirtyExecute` (:62), emits `{hookSpecificOutput:{hookEventName:'SessionStart', additionalContext}}` (:71–78). So AC6.1's "synthesized stdin payload" is **decorative** — the harness's real lever is the **spawn `cwd`** pointed at a planted dirty-EXECUTE fixture tree. Pipe the minimal payload for fidelity/doc, but **do not add stdin parsing** (scope creep) and **do not over-claim** end-to-end coverage.
- **Harness** (`tests/hook-warn-dirty-execute.test.js`, genuinely new — existing `hook-state-write.test.js` tests the *helpers*, not the process): spawn `node hooks/warn-dirty-execute.js` with `cwd` = fixture; assert dirty → stdout JSON contract + exit 0 (AC6.1), clean → empty stdout + exit 0 (AC6.2). Reuse `STATE_EXECUTE_MID_EPIC` + `MILESTONE_E3_CLOSED` string fixtures from `hook-state-write.test.js:30–43,120–123`.
- **`references/hooks-api.md`** (new; mirror `references/state-schema.md` structure): document all three wired events — `PreToolUse(Edit|Write)`→`check-state-write.js` (**reads stdin**, derives baseDir from `tool_input.file_path`), the **unmatched** `SessionStart`→`session-start.sh` (fires on **every** SessionStart), and `SessionStart(resume)`→`warn-dirty-execute.js` (**cwd-driven, ignores stdin**) — call out the **cwd-vs-stdin asymmetry** explicitly. Include stdin shape, stdout/exit-code contract, matcher wiring, fail-open convention, and the **manual real-session procedure** (AC6.4, D-E10-5-ii).
- **The manual leg is essential, not polish:** it's the *only* proof that Claude Code (a) fires the `resume` matcher, (b) spawns with `cwd` = project root (if not, the whole E9 SessionStart net is silently dead), (c) injects `additionalContext`. Frame it that way (A4).

### SessionStart hook contract (external, for hooks-api.md + harness)
- **stdin:** `{session_id, transcript_path, cwd, hook_event_name:"SessionStart", source}` where `source ∈ {startup, resume, clear}` (`compact` documented but UNCONFIRMED). `matcher:"resume"` → fires only when `source==="resume"`.
- **stdout:** `{hookSpecificOutput:{hookEventName:"SessionStart", additionalContext:"…"}}`; **exit 0 required** for injection; SessionStart cannot block. Confirmed by docs + Signal's shipped hook.
- Minimal harness payload: `{"session_id":"test-e10","transcript_path":"/dev/null","cwd":"<fixtureDir>","hook_event_name":"SessionStart","source":"resume"}`.

## Design forks resolved by the planner (plumbing — no user gate)

## Recommended slicing (high-risk-first; pending the 3 gate answers)
