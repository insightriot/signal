## Objective

Solve doc bloat across **all** of a Signal project's docs — permanently — by making finished content *leave* the live files, giving the docs a durable organized structure (read-first index + shallow archive + one-line pointers), and giving existing projects a safe, auto-sensing upgrade path. v0.1.6 shipped the **prevention** half (block new prose pollution in STATE.md + a read-time size banner); this Epic adds the missing **eviction / organization / migration** half.

Design is **borrowed from Curator, owned by Signal.** Curator's model — a read-first INDEX, closed work distilled into short summary cards, a shallow archive tree, a reference/link-health layer, a deterministic engine for mechanical moves with subagents for judgment — is the proven blueprint. Per DECISIONS 2026-07-13, Signal takes **no dependency** on the external Curator CLI; the good parts are reimplemented Signal-native. The installed `auditor`/`distiller` agents + `curator` skill are raw material to study, not runtime dependencies.

> **Re-audit tension, resolved.** The doc-runtime flagship was originally deferred behind the M5 landscape re-audit for one narrow reason: *"indexes and graph/intent-layer must not be designed twice"* (`BACKLOG-REVIEW-2026-07-04.md` §4 Sprint 3). This Epic **folds that one decision in** (FR1) — it makes the canonical doc-organization decision here, grounded in Curator's blueprint, and records it as canonical so the later re-audit **inherits** it rather than re-deciding it. The rest of the re-audit still gates the *speculative* v2 feature ports; it does not gate this.

## Functional Requirements

### FR1 — Canonical doc-organization model (the folded-in traversal decision)
Decide and document the canonical structure Signal uses to organize a project's docs: a read-first index, a shallow archive tree for closed units, short distilled SUMMARY cards for closed work, one-line-pointer eviction from live files, and a lightweight reference/link-health layer. Grounded in Curator's blueprint; recorded as **canonical** (the later re-audit inherits it).
- *AC-seed:* a written model doc exists (`references/doc-runtime-model.md` or equivalent); FR2–FR7 all reference it; no second/competing index or graph design is introduced by any downstream FR.

### FR2 — STATE.md eviction
Make closed-work narrative *leave* STATE.md so the live file stays lean.
- (a) **Migration relocation** — `upgradeStateFile` no longer inlines the entire legacy body; it relocates it to a sibling `.planning/STATE-HISTORY.md` and leaves a one-line pointer. Fixes the acute 455 KB CMMC case at its root (`state.js` ~177 + `writeStateFrontmatter` preservation).
- (b) **Evict-on-close** — at `/sig:ship` (Epic/milestone close) and `/sig:checkpoint`, closed-slice narrative moves out of the STATE.md body to the archive SUMMARY / history file, leaving a one-line pointer.
- (c) **Body skeleton** — a normative "live-above-the-fold" body template in `references/state-schema.md` (resume pointer → in-flight → blockers → pending ops → one-line pointers to closed work), so writer-agents have a structure instead of inventing prose slots.
- (d) **Tier-aware size warning** — surface the size warning in `/sig:status` + `/sig:checkpoint` (extends v0.1.6's flat 150 KB resume banner), with the threshold scaled by PROFILE tier (SKETCH tighter, FULL looser), reading `PROFILE.md` with a flat fallback.
- *AC-seeds:* a migrated legacy STATE.md has its body in `STATE-HISTORY.md` + a pointer, frontmatter unchanged; a shipped Epic's closed narrative is a one-line pointer post-ship; `/sig:status` warns over the tier threshold; Signal's own STATE.md shrinks after the dogfood pass.

### FR3 — FUTURE-IDEAS eviction
Shipped/promoted entries physically leave `FUTURE-IDEAS.md` (to an archive/ledger), leaving the file a lean raw inbox; the `/sig:plan` drain marks disposed entries and evicts them so the file converges instead of accreting (closes the residual behind v0.1.6 FR3, which only stopped disposed entries *re-surfacing*).
- *AC-seed:* a shipped entry no longer appears inline after eviction; a second drain does not re-surface it; file size trends down, not up.

### FR4 — All-docs hygiene runtime
Extend hygiene beyond `.planning/` to the whole doc surface.
- Index freshness + link-health checks wired into the validator/test-suite (a PR that adds a command/agent/skill but forgets the index/map turns the suite **red**) and surfaced at orientation. Absorbs the queued **map-drift-guard** candidate.
- A doc-sweep pass (the `/sig:sweep --docs` concept) reporting stale indexes, dead links, `[FILL IN]` stubs, and drifted rosters across `docs/`, `README`, `CLAUDE.md`, `analysis/`.
- *AC-seeds:* adding an agent without updating the roster/map fails the suite; the sweep reports a seeded stale link + stub; the guard checks roster/counts/version only, never auto-writes prose.

### FR5 — Living BACKLOG.md lifecycle
A groomed, sequenced `BACKLOG.md` supersedes the dated `BACKLOG-REVIEW-2026-07-04.md` snapshot and becomes the single roadmap source; `FUTURE-IDEAS.md` stays the raw inbox; hygiene-vs-roadmap is a **tag inside** the backlog, not a new file; the trigger-watchlist (backlog A1) is evaluated as a drain step so parked items self-monitor.
- *AC-seed:* `BACKLOG.md` is the sequenced source of truth; the dated snapshot is archived with a pointer; the drain evaluates at least one watchlist trigger.

### FR6 — Auto-sensing migrate / re-org command (the high-polish, high-care piece)
A command (name/placement decided in PLAN — its **own** command, e.g. `/sig:migrate-memory`; **not** `/sig:doctor`, which stays install-only) that reorganizes an existing project's docs to the FR1 model.
- **Dry-run by default** — prints the full plan, changes nothing.
- **Git-backed + rollback** — every apply is reversible; refuses to run on a dirty tree without acknowledgement.
- **Move-never-delete** + **dangling-link verification** before and after.
- **Explicit confirm** to apply; **idempotent** (re-running on a migrated project is a no-op).
- Auto-senses the old layout (structural signals + the FR7 stamp).
- *AC-seeds:* dry-run leaves the tree byte-identical; apply is fully reversible via git; no link is broken by a move; a second run on a migrated project changes nothing.

### FR7 — Doc-layout version stamp + upgrade banner
- A `docs_layout_version` integer in STATE.md frontmatter — **its own axis**, distinct from `schema_version` (the frontmatter-format version, M4.5.E6) and from the plugin's release SemVer (same decoupling lesson as Epic-ID vs release tag, D-E11-4). Absent/`1` = pre-reorg legacy; the FR6 command sets it to the current layout version on completion.
- A **SessionStart** check (a small Node hook reusing `state.js`'s frontmatter reader, matching the existing `check-state-write.js`/`warn-dirty-execute.js` Node hooks) plus `/sig:resume` + `/sig:status` emit a one-line banner when a project is pre-reorg on a post-reorg plugin ("docs predate the vX.Y.Z layout — run the migrate command, dry-run first"); post-reorg → silent. Reads the stamp first (fast), falls back to a cheap structural sniff when absent. **Fail-open** — a malformed/absent field never crashes a session. Fires in every installed repo.
- Splits backlog **A4**: install/version drift (plugin vs marketplace pin) stays doctor-adjacent; doc-layout / reorg state leaves doctor for this session-start + resume/status family.
- *AC-seeds:* a pre-reorg project shows the banner once at session start and in resume/status; a post-reorg project is silent; a garbage `docs_layout_version` fails open with no crash and no banner claim.

## Non-Functional Requirements

- **Safety-first (headline NFR, drives FR6).** No data loss; every apply reversible via git; dry-run is the default; move-never-delete; dangling-link verification pre/post. The migrate command is the Epic's one genuinely high-risk surface — these constraints are what make "go big" acceptable.
- **Backward compatibility / no forced migration.** Projects that don't migrate keep working; the banner nudges, never blocks. Both linear and Epic modes handled. No auto-conversion.
- **Signal-native.** No runtime dependency on the external Curator CLI (DECISIONS 2026-07-13). Patterns borrowed; implementation owned.
- **Cross-platform, fail-open hooks.** Node/bash, Windows-safe; hooks never crash a stranger's session (mirrors the E10/E11 hook posture).
- **FULL-tier rigor.** TDD, strict Nyquist (every AC maps to a test), full REVIEW panel. Touches the state-management core every installed repo depends on.
- **Production ops NFRs — N/A.** Health/liveness probe, graceful-shutdown signal handling, structured request logging, security headers (CSP/HSTS/X-Content-Type-Options), rate limiting: **N/A** — this is local doc-tooling / CLI + hooks, with no network-service surface. Recorded explicitly per the FULL-tier NFR prompt.
