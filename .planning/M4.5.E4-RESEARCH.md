## 1. R1 — RESOLVED: swap storage to Node's built-in `node:sqlite`

R1 (the runnable example's native-dependency caveat) was the one open item DISCUSS handed to PLAN. **Resolution: option (a) — swap `better-sqlite3` → `node:sqlite` (`DatabaseSync`).** Confirmed by the phase/risk agent:

- Storage is isolated in **one 32-line file** (`src/storage.js` — the only file importing `better-sqlite3`). `service.js`/`server.js` already take `storage` by dependency injection; `service.test.js` already fakes the interface. So the swap is **near-mechanical**: change the import + constructor, move 3 pragmas from better-sqlite3's `.pragma()` into `db.exec('PRAGMA …')`. Effort **S**.
- `node:sqlite` is a ~1:1 API match (`exec`/`prepare`/`run().changes`/`get` all carry over). Persistence semantics preserved, so the persistence tests (the example's showcase coverage) stay intact.
- **Removes the only runtime dependency** — satisfies AC9 ("no new runtime deps") by *removing* one; aligns with Signal's anti-over-engineering ethos.

**Environment check (PLAN Step 6, run concretely):**
- Dev machine: Node **v22.13.0**, npm 10.9.2, macOS arm64, Xcode CLT present.
- The *committed* dogfood tree currently **fails 22/39 tests** due to native-ABI drift (stale `node_modules` binary built for `NODE_MODULE_VERSION 141`, machine needs 127) — this is A2 failing for the stale tree. But `node_modules` is gitignored, so a stranger runs a fresh install.
- A fresh `better-sqlite3@^12.9.0` install *does* fetch a working prebuilt on this Mac — so "keep it" (option b) would work on common platforms, but carries the ABI footgun off the prebuilt happy path + a deprecated `prebuild-install`.
- `node:sqlite` is available on this machine: `require('node:sqlite')` works flag-free on 22.13.0 (emits an `ExperimentalWarning`). It is flag-free + RC-stable on Node 24+.

**Plumbing decisions locked (product-level approval already covers "make it runnable"; these are implementation defaults):**
- **Node floor: `engines.node >= 22.5`** (first version with `node:sqlite`), with the example's `test` script invoking `node --experimental-sqlite` so the suite runs clean on the actual dev machine (22.13.0). The example README notes Node 24+ needs no flag. Rationale: verifiable on *this* machine today; widest compatibility; the experimental warning is cosmetic for a demo. (If Node 24 is readily available at EXECUTE, bumping to `>=24` for a flag-free run is a free upgrade — left to EXECUTE's discretion.)
- One test to re-verify after the swap: `tests/shutdown.test.js:63-79` (the "DB_PATH is a directory → non-zero exit" case) — confirm `DatabaseSync` still throws on a directory path.

Sources: `node:sqlite` added in Node 22.5.0 (flag-gated, Stability 1.1); flag-free RC (1.2) in Node 24 — [nodejs.org v22 docs](https://nodejs.org/docs/latest-v22.x/api/sqlite.html), [v24 docs](https://beta.docs.nodejs.org/sqlite.html).

## 3. Guard test — critical design constraint (FR4/AC4)

From codebase agent #1:
- **`readState` does NOT throw on a legacy STATE.md** — it returns `{ …, _schema: 'legacy' }`. So a guard that only checks "doesn't throw" would **silently pass the un-refreshed stale file and defeat its own purpose.** The guard MUST assert **`state._schema === 1`**.
- `readProfile` validates strictly and already passes on the example's PROFILE.md today.
- Placement: `tests/example-currency.test.js` (auto-discovered by `vitest.config.js` include `tests/**/*.test.js`). Read-only — no fixture copy needed (unlike `state-schema.test.js`, which copies because it mutates).
- Path pattern: `const EXAMPLE_DIR = join(__dirname, '..', 'examples', 'url-shortener')` → `readProfile(EXAMPLE_DIR)` / `readState(EXAMPLE_DIR)` (both append `.planning/` themselves). Template files: `tests/state-schema.test.js:21-38`, `tests/profile.test.js:1-12`, `tests/state.test.js:6`.

**Validator (FR8):** `tools/validate-plugin.js` stays green with no change (its `REQUIRED_*` lists don't mention `examples/` or `docs/`, and it never scans them). **Plumbing decision:** add `docs/vs.md` to `REQUIRED_FILES` (error-level) for cheap belt-and-suspenders against accidental deletion — there's test precedent (`validate-plugin.test.js:102-109` runs the validator as a subprocess and asserts exit 0; the file must exist before that stays green, which Slice 2 guarantees). `examples/` is **not** gitignored (only `.dogfood/` is) — committing into a tracked `examples/` + the guard is the structural fix for *why the dogfood rotted*.
