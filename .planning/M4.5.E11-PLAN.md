# M4.5.E11 — Plan (PLAN phase)

**Goal (one sentence):** Make Epic mode first-class — commands open/track Epics, auto-write a strict-shaped `current_epic`, name artifacts `{EpicID}-*.md`, and honor a per-Epic tier — all **additive** over a byte-identical linear mode.

**Tier:** FULL / `gate_strictness: strict`, `nyquist_enforcement: strict`, `plan_validation_dims: all`. Tasks commit atomically as `M4.5.E11.S{N}.t{K}`; checkpoint per slice.

## Architecture decisions (locked this phase)

- **Epic-ID shape = M-shaped only** (D-E11-4): `current_epic` is always `/^M\d+(\.\d+)*\.E\d+$/`; version numbers are separate release tags. One **canonical strict regex** shared across `state.js`/`retrospective.js`/`milestones.js` (kills the regex schism).
- **Epic-creation UX = `--epic <name>` flag** on `/sig:discuss` + `/sig:new-project` (no new required command → no `REQUIRED_COMMANDS`/install-contract change). A dedicated `/sig:epic` is a FUTURE-IDEAS follow-on only if an explicit abort-without-opening-next is ever needed.
- **Close = roll-on-open, never clear to null.** Opening the next Epic rolls `current_epic` (atomically resetting `current_wave: null` + `current_tasks: []` — this also covers abandon). Done-signal = `{EpicID}-RETROSPECTIVE.md` existence (NOT `phase=SHIP` — Signal never moves SHIP into `completed_phases`). A writing command against a *done* Epic with no `--epic` **errors, requiring `--epic`** (no clobber).
- **Per-Epic calibration = `.planning/{EpicID}-PROFILE.md`, whole-file shadowing** (B). Effective profile = Epic file if it exists (with strict `current_epic`), else project `PROFILE.md`. No merge (PROFILEs are complete "not by reference"), no STATE field, no `schema_version` bump. New `readEffectiveProfile(baseDir, currentEpic)`.
- **Mode signal = non-null `current_epic`** (C), canonical predicate `typeof current_epic === 'string' && current_epic.trim() !== ''`, fail-open to linear on anything empty/malformed.
- **Retro-gate = two-tier** (D-E11-5, refined by validation B2). There are two retro-enforcement paths: the **hook** (`checkProposedStateWrite`, fires on arbitrary STATE edits — the stranger blast-radius surface) → **warn, non-blocking** (exit 0) + throw-safe; the **command gate** (`shipFR1Check` via `ship.md` §0.5, fires only when you run `/sig:ship` on an Epic close) → **retained hard gate** (E9's SHIP contract — running the command opts you into its documented retro step). Rationale: hooks nudge on process (Brett's principle), but invoking the workflow command = opting into its contract. **Surfaced at the plan gate** — if `/sig:ship` itself should also downgrade to warn, that's a one-line change.
- **FR2 write-half is prose → extract `artifactName()`** so the branching logic is RED-first testable; markdown prose covered by prose-review + `{phase}-VERIFICATION.md` attestation (E10.S1.t4 pattern).

## Slices (vertical, high-risk-first)

### S1 — `current_epic` write-half + mode detection + back-compat net (FR1 + FR4) — RISK: HIGH — first
*Goal:* a command opens/rolls an Epic; `current_epic` is auto-written + strict-validated; mode-detection is canonical; linear stays byte-identical; the SHIP retro path warns (not blocks) and is throw-safe.
- **t1** — Canonical strict Epic-ID **validation** regex `EPIC_ID_STRICT_RE = /^M\d+(\.\d+)*\.E\d+$/`, defined once in `state.js` and *reused for validation* by `retrospective.js`'s `deriveRetroPath`. **Precision (C1):** `milestones.js:20` `CURRENT_EPIC_RE` is a *capturing, unanchored* regex that **extracts** the milestone number — it is NOT replaced; it stays a derive fed a *pre-validated* ID. Pin the writer's real output shape to depth-2 `M{maj}.{min}.E{n}` (what `currentMilestone` captures) so the round-trip AC tests real IDs; `EPIC_ID_STRICT_RE` permits depth-3 but the writer never emits it. *AC:* validator accepts `M4.5.E11`/`M5.E1`, rejects `v0.1.6`/`''`/`../x`; every ID the writer emits round-trips through `currentMilestone` to a non-null milestone. *Test:* unit (validator + `currentMilestone` round-trip on real shapes). **S**
- **t2** — `setCurrentEpic(baseDir, epicId)` in `state.js`, cloned from `setCurrentTask` (`withStateLock`→`readStateForMutation`→`stripStateMeta`→set→`writeStateFrontmatter`). Receives the *already-resolved* strict ID from t7; re-validates against `EPIC_ID_STRICT_RE` **before** disk (defense-in-depth); on open/roll resets coupled `current_wave: null` + `current_tasks: []`; idempotent. **(C3)** `clearCurrentEpic` is NOT built — the locked "roll-on-open, never clear" semantics give it no caller (YAGNI; a future abort-Epic feature adds it then). *AC:* valid id round-trips; invalid id throws pre-disk; coupled fields reset on roll; other frontmatter preserved; writing the already-set id = safe no-op. *Test:* unit (mirror `setCurrentTask` suite). **M**
- **t3** — `detectMode(state)` → `'epic'|'linear'` pure fn (canonical predicate; fail-open to linear on empty/malformed, never throws). *AC:* `null`/absent/`''`/whitespace → linear; `M4.5.E11` → epic. *Test:* unit. **S**
- **t4** — Harden the E9 retro-gate **hook** path (`hooks/check-state-write.js:103` → `checkProposedStateWrite` at `retrospective.js:358`): (a) try/catch → fail-open exit 0 on any throw (R2, confirmed reachable — `deriveRetroPath` throws on malformed `current_epic`, call is currently unguarded); (b) missing-retro-at-Epic-SHIP on this hook path → **non-blocking warn** (exit 0 + stderr), never exit-2 block (D-E11-5). Verify current behavior first. **Scope note (B2):** this is the *hook* only (fires on arbitrary STATE edits — the blast-radius surface). The *command-layer* gate `shipFR1Check` (`retrospective.js:618`, invoked by `ship.md` §0.5) is a **separate, intentionally-retained** hard gate — see the two-tier split in Architecture decisions + S4.t3. *AC:* malformed `current_epic` SHIP write → exit 0; hook-path Epic-SHIP w/o retro → exit 0 + warning in stderr (never exit 2). *Test:* hook integration (spawn, assert exit code + stderr). **M**
- **t5** — Done-Epic guard: `isEpicDone(baseDir, epicId)` (retro file exists) + command-level rule — writing command against a done Epic with no `--epic` errors requiring `--epic`. *AC:* done Epic + no flag → error; done Epic + `--epic new` → rolls; live Epic + no flag → proceeds. *Test:* unit (`isEpicDone`) + prose/attestation for the command rule. **S–M**
- **t6** — Back-compat net: linear golden fixture `tests/fixtures/epic-native/linear/.planning/` (null `current_epic` + `1-*.md`); byte-identity triad (`artifactName` null-case, `readProfile` no-Epic golden, full 894-suite green with no fixture edits). *AC:* linear behavior byte-identical to pre-E11. *Test:* golden + regression. **S**
- **t7** — `--epic` command wiring + Epic-ID assignment — the FR1 *command surface* the validator flagged as homeless (B1). `deriveNextEpicId(baseDir, {milestone})` pure helper: scan existing `M*.E*-*.md` artifacts + STATE for the max `E{N}` under the milestone, return the next strict ID. Parse `--epic <name>` in `commands/discuss.md` + `commands/new-project.md` with **derive-by-default, explicit-override**: if the arg is itself an `EPIC_ID_STRICT_RE`-valid ID (e.g. `M5.E1` opening a new milestone) use it verbatim; otherwise treat it as a human label and derive `M{current-milestone}.E{next}`; then call `setCurrentEpic` (t2) + record the label. This resolves the fork — commands DO assign IDs (D-E11-1 "no hand-typing"); the CLI arg is a label, the STATE value is machine-assigned. *AC:* `--epic <label>` on `/sig:discuss` opens an Epic whose auto-assigned strict ID lands in `current_epic` (nothing hand-typed into STATE); an explicit strict-ID arg is used verbatim; `deriveNextEpicId` returns `M4.5.E12` when `M4.5.E11` is latest. *Test:* unit (`deriveNextEpicId`) + prose/attestation for flag parsing. **M**
