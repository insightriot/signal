6. The Missing Layer — Rigor Calibration (Phase 0)

Every repo treats every project with the same rigor. That's wrong. A marketing homepage ≠ Conversor. /sig:calibrate at project kickoff tunes the system and routes the workflow into one of four modes. Calibrate, not triage — this is instrument-tuning before signal detection, not emergency stabilization.
Novelty ▲ Novel  ·  Familiar ▼
NOVEL × HIGH STAKES
SPIKE
Novel problem with real stakes. Heavy validation, lighter verification, time-boxed.
Phases: Ideate → Validate → Spec → Execute → Verify → Ship
Example: Early AI product exploration, novel market entry
FAMILIAR × HIGH STAKES
FULL
Production app. Real users. Meaningful blast radius. All guards on.
Phases: All 10 (Ideate → Compound)
Example: Conversor, auth system, payments, SaaS
NOVEL × LOW STAKES
SKETCH
Throwaway, demo, exploration. Fast iteration, visual verdict only.
Phases: Ideate → Execute → Visual Check
Example: Homepage, landing page, demo, prototype
FAMILIAR × LOW STAKES
FEATURE
Adding to existing codebase. Skip upstream (problem already validated).
Phases: Spec → Plan → Execute → Verify → Review → Ship
Example: New feature in known app, enhancement, bug fix
← Stakes (blast radius if wrong) →

The Calibration Mechanism

The /sig:calibrate command asks 5 questions and writes a profile that every downstream phase reads.
The 5 diagnostic questions
  1. Scope — net-new or modifying?
  2. Stakes — blast radius if broken?
  3. Novelty — known analogs or new territory?
  4. Reversibility — two-way door test
  5. Horizon — throwaway or durable?
Output: .planning/PROFILE.md
tier: SKETCH
stakes: low
novelty: familiar
phases_skipped:
  - validate
  - review
  - cso_security
  - compound
rigor_overrides:
  tdd_required: false
  security_audit: false
escape_hatch: /sig:escalate
Key principle — phases aren't removed, they're gated. /sig:escalate promotes a SKETCH to FULL mid-flight if scope grows. Prevents the "cheap mode becomes tech debt" failure where something you prototyped ships, becomes real, and has no retroactive rigor.

Closest analogs in the landscape — none fully solve it

GSD gsd-spike / gsd-sketch escape hatch, not classifier
Light-mode commands, but operator must already know to pick them. No diagnostic step.
gstack /plan-ceo-review 4 scope modes plan-time, not project-time
Expansion / Selective / Hold / Reduction modes run at plan-review time, not project kickoff.
OMC deep-interview 20% ambiguity gate spec clarity ≠ project complexity
Measures spec ambiguity. Orthogonal to stakes/novelty — you can have a clear spec for a throwaway.
Strategic implication for your consultancy: Calibration is the feature that makes AI-assisted dev *adoptable* by mid-market orgs. Without it, they face "full rigor always" (simple stuff feels overbuilt) or "light touch always" (important stuff ships broken). Signal earns the right to prescribe rigor — by showing it knows when *not* to.
